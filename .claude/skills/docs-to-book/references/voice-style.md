# 文风基准

全书文风：**沉静陈述体**。像一本克制的工程笔记，不是营销文案，不是口语对话。目标是接手者能安静地读懂，而不是被语气吸引。

## 三条总则

1. **陈述，非规划**：写「系统是怎样的」，不写「系统将要做」「我们打算」
2. **事实保真**：数字/表名/字段/命令/配置值逐字保留，只润色散文
3. **克制**：去夸张、去口语、去感叹、去稀释虚词

## 去 emoji（硬规则）

| 类型 | 处理 | 例子 |
|---|---|---|
| 装饰性 emoji | **移除** | 💡⚠🚨📌🎯🔐→ 删 |
| callout 图标 | 移除 emoji，改 CSS 文字标签 | 「💡 提示」→「附注」「建议」「留意」 |
| 状态 emoji | 转文字/几何字符 | ✅→✓、❌→✕、⭐→★ |
| 几何/功能字符 | **保留** | ✓ ★ ◐ ☰ ⌘ ↵ |

callout 用 CSS 注入文字标签，结构示例：

```html
<div class="callout note">
  <p>...</p>
</div>
<!-- CSS 自动给 .note 加「附注」标签，无需手写图标 -->
```

## 标点规范

- CJK 相邻的半角 `,;` → 全角 `，；`
- **保护** code / pre / mermaid 块内的标点不动（代码里半角是对的）
- 用脚本批量转换，不要手动改（容易漏、容易改错代码）

转换脚本思路（保护代码块）：

```python
import re, glob
for f in glob.glob('content/*.html'):
    text = open(f).read()
    # 按块切分，code/pre/mermaid 块整体保护
    parts = re.split(r'(<(?:pre|code|div class="mermaid")[^>]*>.*?</(?:pre|code|div)>)', text, flags=re.S)
    for i, p in enumerate(parts):
        if i % 2 == 1:  # 奇数索引 = 被保护块
            continue
        # CJK 旁的半角逗号转全角
        p = re.sub(r'([一-鿿]),', r'\1，', p)
        p = re.sub(r'([一-鿿]);', r'\1；', p)
        parts[i] = p
    open(f, 'w').write(''.join(parts))
```

## 措辞对照表

| 避免（口语/营销/规划） | 改成（沉静陈述） |
|---|---|
| 将要做 X / 我们打算 | 系统做 X |
| 带你了解 / 带你进入 | （删，直接陈述） |
| 其实 / 事实上 / 那么 | （删） |
| 接下来我们看 | （删，直接写） |
| 对不对 / 是不是 | （删，陈述即可） |
| 拍脑袋 / 砸下来 | 凭直觉 / 降临（或改正式表达） |
| 极致 / 强大 / 硬核 | （删或换精确词） |
| 那是下游系统的事 | 由下游系统负责 |
| 长成今天的样子 | 演进至今 |
| ！（感叹号） | 。（句号） |

## 润色时的硬约束（给子代理）

委派子代理润色时，prompt 里写死这几条：

```
1. 只改散文措辞与语气，不改任何事实
2. 数字、阈值、表名、字段名、workflow 名、命令、配置值 = 逐字保留
3. 不动 HTML 结构、不动 mermaid 块、不动 code/pre 块
4. 去口语/营销/感叹，改沉静陈述体
5. 去装饰 emoji，保留几何/功能字符
6. CJK 旁半角标点转全角（code 块内不动）
7. 交付物：只返回「slug + HTML 正文」文本，不自己写文件。
   文件落盘由主 agent 统一执行，避免并发写入冲突。
```

### 为什么子代理不自己写文件

多人同时 Write 到同一目录时不可避免地会出现冲突——主 agent 在写 `rule-engine.html`，子代理也在写同一个文件，一方报错。统一由主 agent 落盘的方案简单可靠：子代理产出纯文本，主 agent 负责最终的文件 I/O。这样不存在文件锁问题和覆盖问题。

## 验证

润色后跑一次全局 grep，确认无残留口语词：

```bash
grep -rnE "带你|其实|那么我们来|对不对|极致|拍脑袋|！" content/
```

命中 0 为干净（注意排除 code 块内的合法感叹号）。

## 反模式

- ❌ 为了「文采」加形容词、比喻（工程文档要精确，不要文学化）
- ❌ 第二人称「你」泛滥（除阅读指南这类导航页可适度用）
- ❌ 把「待办/规划」润色成了「已完成」的语气（事实保真）
- ❌ 手动改标点漏了 code 块（用脚本保护）
