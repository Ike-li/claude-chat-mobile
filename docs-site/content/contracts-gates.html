契约与门禁把「脑子里的分层约定」变成 npm run check 硬闸：事件名与类型不许漂移，import 不许缠死，仓库清单不许悄悄回堆。

node scripts/contract-check.js
node scripts/doc-consistency.js

步骤职责
ESLint语法与基础风格、死代码、未定义引用
check-import-boundaries分层边界 + 零循环依赖
contract-check出向事件类型与入向 socket 事件双向保真
doc-consistency残留文档/注释与约定一致性抽查
visual mock registry视觉 mock 场景注册面 ⊆ 契约
playwright forbidden禁止脆弱选择器/反模式
inventory:check仓库文件分类与生成物不过期

import 边界：BOUNDARY_RULES

扫描根为 src/、public/js/ 与 server.js。五条规则：

规则名不变量
frontend-no-backendpublic/js 不得 import src/
backend-no-frontendsrc/、server.js 不得 import public/js（共享白名单除外）
shared-is-leafsrc/shared 不得反向 import 其他后端域
server-is-sink只有 server.js 与 src/server 自身可 import src/server
runtime-no-tooling运行时源码不得 import scripts/ 或 tests/

另做零循环依赖检测（含动态 import() 扫描，不能靠动态 import 绕边界）。

SHARED_ALLOWLIST =
  public/js/canonicalize.js  // 与指纹规范化共用
  public/js/logic.js         // 前后端共用纯决策声明；单测直接覆盖
建议新增「前后端都要引用」的逻辑时，优先放进已有白名单文件的纯函数，或先扩展门禁白名单并写清理由；不要让 public/js 直接 import src/。
附注tests/ 与 scripts/ 不设 import 边界（工具与测试可跨域引用），但运行时源码不得反向依赖它们（runtime-no-tooling）。

事件契约

权威列表在 scripts/agent-event-contract.js，由 contract-check 与源码/前端/mock 注册面交叉比对。

  • 出向：agent:event 的 type 必须在 AGENT_EVENT_TYPES allowlist（真实路径与 visual mock 对照）。
  • 入向：客户端 emit 的 socket 事件名必须在 INBOUND_SOCKET_EVENTS；server 注册面与契约双向相等。
  • 前端 emit 面 ⊆ 契约；visual mock ⊆ 契约。漏注册或多注册都会红。

完整列表见事件与端点参考。

Inventory

inventory:update
node scripts/repo-inventory.js --write，有意增删文件后更新清单与 docs/repository-map.md
inventory:check
node scripts/repo-inventory.js --check，拒绝未分类文件与过期生成物

新文件进入仓库前须能归类；审计笔记、一次性提案等不应悄悄回到主树。check 红时先看是契约漂移还是 inventory 未更新。

建议推送带 CI 的仓库前，可用 CI=true npm test 模拟 CI 行为。
