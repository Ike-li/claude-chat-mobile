控制面状态、CLI transcript、附件与内存态分属不同根目录。备份与清盘前，先分清「可迁出的 CCM 状态」与「CLI 真相源」。

存储全景

路径角色生命周期
$CCM_DATA_DIR/sessions.json会话元数据与偏好（含隐藏列表等），非消息正文持久；宜 0600
…/approval-requests.json审批台账；执行闸门仍在内存持久台账
…/audit-records.json安全审计环形记录持久
…/trusted-devices.json · pending-devices.json设备 TOFU持久；清盘 = 废掉手机信任
…/cf-access-certs.jsonAccess JWKS 缓存可弃；损坏再拉
…/init-cache.json模型列表等 init 缓存可弃
…/push-subscription.jsonWeb Push 订阅持久
workdirs.json（由 WORK_DIRS_FILE 指向）多工作区白名单；可热加载不在 CCM_DATA_DIR；勿提交
~/.claude/projects/<编码cwd>/<sessionId>.jsonl消息真相源；与 CLI /resume 同源不随 CCM_DATA_DIR 迁走
WORK_DIR/.ccm-uploads/附件原件；prompt 注入绝对路径删附件 → 历史预览失效
内存 ring buffer · agents Map最近 2000 事件、在途状态易失
默认根CCM_DATA_DIR 未设时回落仓库 ./data。生产建议外置绝对路径、目录 0700、敏感 JSON 0600。多数状态模块在 import 时解析路径，故 env 须在 server.js 薄入口加载完成后再装 app.js。

双真相源

CCM 控制面

sessions.json、设备、审批台账、推送、缓存。描述「web 如何看待会话与谁可连」，不替代对话正文。

CLI transcript

~/.claude/projects/**/*.jsonl 是对话与工具轨迹的权威落盘。历史冷读、镜像追平、续接都读这里。

风险只备份 CCM_DATA_DIR 会丢掉消息正文；只备份 ~/.claude 会丢掉设备信任、隐藏会话与推送订阅。两端不一致时可能出现「有元数据、无历史」或续接失败。

备份含义

  • 完整可恢复：外置 CCM_DATA_DIR + 相关 ~/.claude/projects/ + 各 WORK_DIR/.ccm-uploads/（若仍要打开历史附件）+ .env / 隧道凭据（另管）。
  • 可丢弃：init-cache、cf-access-certs、环形缓冲与 agents 内存态。
  • 迁移：停常驻服务 → 拷贝数据根 → 改 CCM_DATA_DIR → node scripts/doctor.js → kickstart 重启。
热加载例外WORK_DIRS_FILE 指向的文件由 server 监听，改完即生效、免重启；被移除目录上已开会话可继续，仅拒新开。逗号串 WORK_DIRS 无文件可 watch，改后须重启。
危险一次性测试 server 必须隔离 PORT / WORK_DIR / CCM_DATA_DIR / AUTH_TOKEN，否则会污染生产 data。

硬编码证据：src/sessions/history.js 中 CLAUDE_DIR = join(homedir(), '.claude', 'projects')；deployment 文档亦写明 transcript 不迁入 CCM_DATA_DIR。
