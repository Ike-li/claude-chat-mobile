通知与状态栏都是「尽力增强」：失败不阻断主对话；Web 驾驶与 CLI 驾驶的状态来源不同。

Web Push

  • 需 VAPID_PUBLIC_KEY / VAPID_PRIVATE_KEY / VAPID_SUBJECT 三项齐全，否则优雅缺席。
  • 订阅保存在 $CCM_DATA_DIR/push-subscription.json。
  • HTTP：GET /push/vapid-public-key、POST /push/subscribe（均走 httpAuth）。
  • 典型触发：审批、提问、结果类（经分类与节流）。
  • 实现：src/ops/notify-channels.js + notifications.js。
留意iOS 上 Web Push 要求「添加到主屏幕」且 https。局域网 http 不可用；此时应配 ntfy 或保持页面前台。

ntfy

  • NTFY_URL + NTFY_TOPIC 启用；可选 NTFY_TOKEN 访问私有实例。
  • PUBLIC_URL 用于通知深链绝对 URL；空则回退 https://$CF_ACCESS_HOSTNAME。
  • server 一行 POST 到 ntfy；手机装 ntfy app 订阅 topic 可锁屏收通知。
  • 正文可能含命令 / 审批摘要 → 务必自托管或私密 topic + token，勿用公共 ntfy.sh 裸 topic。

节流窗口：NOTIFY_THROTTLE_MS 默认 60000（同会话同类通知）。投递失败会记入服务健康（deliveryFailure），面板可展示，不阻断对话。

Statusline 来源选择

场景来源
Web 驾驶SDK 状态（模型、上下文、成本等）
CLI 驾驶 + bridge 已装CLI 快照目录中的 statusline 观察态
WEB_STATUSLINE=off关闭 Web 状态栏 UI
CLI_STATUSLINE_BRIDGE=offserver 侧回滚为 SDK-only（不自动卸载 CLI wrapper）
 
npm run statusline:status      # 只读查看，不改 ~/.claude
npm run statusline:install     # 显式安装；不随 npm install / start 自动执行
npm run statusline:uninstall

安装后须重开 Claude CLI 并重启常驻 server。CLI 与 server 的 CLI_STATUSLINE_DIR 须一致（默认约定在 ~/.claude/ccm/… 下）。

建议多数用户只需 Web 驾驶的 SDK 状态栏。仅在「Web 只读镜像正在 CLI 里跑的会话、且要看 CLI 同源 statusline」时再装 bridge。

动态状态行与回合收尾

  • 运行中可展示 CLI 式动态状态行（动词 / 秒表 / tokens 等，经 status_line 事件）。
  • 成功轮可有回合收尾行（对齐 CLI：过去式动词 + 时长）；中断/错误路径有意不同文案。

服务状态面板

instances 广播可带判定化字段：

service: {
  startedAt,
  deliveryFailure,
  rateLimitLockout,
  clientError
}
  • 面板渲染「基础 + 判定化告警」两段；告警带 24h 时效窗自动退场。
  • 裸计数器不进入面板（对人无参照系）；原始计数留 /metrics。
  • 与「需要你 (N)」角标聚合是不同轴，不混判。
