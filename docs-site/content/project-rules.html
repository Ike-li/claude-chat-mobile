分支、门禁、测试分层与贡献约定。遵守这些约束，比多写一层文档更能保持仓库可合并。

分支与 worktree

  • 日常开发在 dev；master 为稳定 / GitHub 默认 / 分支保护。功能完成后由 dev ff 合并 master，发版用 scripts/release.sh。
  • 仓内 .worktrees/ 是其他分支（promo、gh-pages）的检出位，不是本分支源码。搜索与审查排除该目录（裸 grep/find 加 --exclude-dir=.worktrees）。
  • 宣传物与 landing 站留在 gh-pages / promo 孤儿线，不回灌 master/dev。

门禁

npm run check
# ESLint
# + check-import-boundaries（分层 + 零环依赖）
# + contract-check（出向 agent:event + 入向 socket）
# + doc-consistency
# + visual mock registry / Playwright 禁止模式
# + inventory:check

新增、移动、删除入库文件后须 npm run inventory:update，否则 check 红。生成的 docs/repository-map.md 勿手改正文。

建议推送带 CI 的仓库前可用 CI=true npm test 模拟 CI。集成在 CI 中常 skip 需 CLI 段；真 Claude turn 用 RUN_CLAUDE_INTEGRATION=1。

测试分层

命令含义
npm run test:unit纯逻辑单测，零 token，最快
npm run test:integration起真 server；需本机 CLI
npm testunit + 可靠 integration；--test-force-exit
npm run test:e2ePlaywright 移动端 UI，mock server，零外部 token
npm run test:smoke真 Claude 回合；默认不跑，耗 token

E2E 断言基于 DOM 状态，非像素比对。mock 发出的事件 type 必须 ⊆ 契约 allowlist。

工程约定

  • 不引入 TypeScript 编译闸（tsc）：项目以 Node ESM + ESLint + 契约脚本为门禁。
  • src/server/app.js 组装根有意保留：mirror / catchUp / openInstance 与契约路由共享顶层可变状态；勿为「拆文件」而拆同步引擎，除非有明确边界设计与全量回归。
  • commit 不加 Claude 署名：不要 Co-Authored-By / Claude-Session 行；说明改了什么、为何、如何验证即可。
  • 一次性测试 server 须隔离四件套：PORT / WORK_DIR / CCM_DATA_DIR / AUTH_TOKEN，避免污染生产 data。
  • 生产改 .env/代码后须重启常驻进程；例外是 WORK_DIRS_FILE 热加载。
附注并发开发同一文件时用 git worktree 隔离，避免互相 reset 冲掉未提交工作。本手册产物在 gh-pages 的 docs-site，不回写 dev 的产品 docs。
