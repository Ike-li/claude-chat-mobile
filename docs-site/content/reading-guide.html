读者打开手册后应立刻知道：怎么读、读哪条线、按什么顺序。主线兜底，速览与角色线是捷径。

主线通读

主线 = 七部分的天然顺序，与侧栏一致：

起点 → 认识 → 演进 → 方法论 → 实现 → 数据 → 运维 → 规范

预计 2–3 小时。适合第一次完整接手。每页可独立读懂，但仍建议按序建立心智模型。

30 分钟速览

  1. 封面与总览。封面 → 项目总览：一句话定位与场景。
  2. 架构与边界。系统架构 → 职责边界：分层与「不做什么」。
  3. 方法论三页。终端等价 → 安全模型 → 单驾驶员。
  4. 能跑起来。快速开始；若要公网再扫生产部署标题。
  5. 校准事实。漂移清单：文档与代码不一致处以代码为准。
