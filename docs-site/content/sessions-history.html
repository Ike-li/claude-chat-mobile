会话有两层真相：Web 元数据在 sessions.json；消息正文在 CLI transcript jsonl。混淆两者是接手时最常见的误操作来源。

双真相源

层路径内容
元数据 SoT$CCM_DATA_DIR/sessions.jsoncurrentByCwd、标题、model/effort/permissionMode、隐藏列表、活动时间等
消息 SoT~/.claude/projects/<encoded-cwd>/<sessionId>.jsonl完整 transcript；与 CLI /resume 同源
以代码为准改 sessions.json 不会改聊天内容。备份 CCM_DATA_DIR 不会带走对话正文；清仓库 data/ 可能丢设备信任与指针，但 CLI 历史仍在。

列表与冷读

  • 列表优先 SDK listSessions，失败回落扫盘；结果可与 sessions.json 元数据增强合并。
  • 历史回显走鉴权 socket 事件 session:history，不开无鉴权 HTTP 数据端点。
  • 冷读上限：HISTORY_MAX_MESSAGES = 2000。
  • CLI 系统噪音行经过滤（isMeta / 系统行启发式），避免污染镜像与尾部判定（如镜像锁释放、尾部 pending）。
  • resume 时可冷读会话末条 assistant 模型作 chip 回落展示（不写入 activeModel 以免误重置）。
留意getSessionMessages 返回的是清洗过的 message，不是 JSONL 原行。同步引擎（catch-up / mirror）必须读 jsonl 路径，不能用该 API 当事实源。

打开路径：FRESH vs resume

路径触发档位来源要点
FRESHsession:new；空首页首条消息无 resumeId不继承 cwd 末实例 permission/effort；来自 CLI settings 默认与 UI pending
resume列表切入；懒开有 saved.idsessions.json → transcript 末 mode/model → 继承逻辑；先 prepareSessionForWebResume 释放 CLI bg 锁
  • 启动时不自动 resume；空首页手选会话。
  • 懒开：首条消息且无 live 实例时 dedupedResume（single-flight）。
  • idle reclaim：完全空闲超过 INSTANCE_IDLE_RECLAIM_MS（默认 1800000，0=关）可回收子进程；盘上 transcript 保留，再发会 resume。

删除分层

  • L1：hide（写入隐藏列表），transcript 仍在，列表不可见。
  • L2 永久删除：动文件；受安静期等守卫（SESSION_DELETE_QUIET_MS 默认 300000）保护，避免误删终端仍在用的会话。

worktree 会话

worktree-sessions.js 发现 git worktree 内会话，使 Web 可触达「在 worktree 里跑的 CLI 会话」——终端「cd 进 worktree 即 /resume」的 web 等价方向。实现与 knownWorktrees 注册、列表分组相关（以当前 dev 接线为准）。
