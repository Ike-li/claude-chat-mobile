系统是一条默认上锁的转发链：手机 UI 经 Socket.IO 到本机 server，再经 Agent SDK 驱动 claude CLI 读写你的工作区。

后端域划分

目录职责
src/server/组装根（sink）：HTTP 壳、Socket 契约、实例生命周期与业务接线
src/agent/SDK 桥、审批台账、消息去重、模型缓存、CLI mirror
src/auth/CF Access JWT、设备 TOFU、鉴权限速
src/sessions/会话元数据、transcript 历史、工作区白名单、worktree 发现
src/files/上传、只读浏览/预览、路径闸
src/ops/审计、指标、推送、statusline、doctor
src/shared/叶子：日志时间戳、脱敏、串行写盘

src/server 是唯一组装 sink：只有 server.js 与 src/server/* 可 import 它。域模块不得反向依赖 server。边界由 scripts/check-import-boundaries.js 在 npm run check 中硬闸。

消息流程（五步）

  1. 发送。手机 user:message {text} → server 校验 → 路由到 agents.get(instanceId)（懒开 FRESH 或 resume）。
  2. 驱动。文本进入 AgentSession 的 streaming input → SDK → CLI 在 WORK_DIR 执行。
  3. 映射。SDK 消息经 map()：流式文本 → text_delta；工具 → tool_use / tool_result；白名单外 → permission_request。
  4. 信封。每个事件套 {seq, epoch, sessionId, instanceId, cwd, ts, type, payload} → 入 2000 条环形缓冲 → io.emit（前端按 viewingInstanceId 分流；后台 tab 的高频 delta 不广播以省带宽）。
  5. 补发。断线重连 sync:since {lastSeq} 从缓冲回放；epoch 变化表示服务端换了实例，客户端重置去重基线。

启动路径

server.js
  loadRuntimeEnvironment()   # dotenv；ANTHROPIC_* 只认 shell
  installLogTimestamps()
  dynamic import src/server/app.js
    initCfAccess / parseServerConfig
    preflight（WORK_DIR、CLAUDE_BIN、workDirs）
    Express + Socket.IO + 鉴权 + instance manager
    expireOrphanedPending + 审批留存清扫
    listen(port, AUTH_TOKEN ? 0.0.0.0 : 127.0.0.1)
留意无 AUTH_TOKEN 时只绑定 127.0.0.1，手机与隧道均不可达。这是有意设计，不是故障。

前端结构

  • public/js/app.js：大壳，socket 与 UI 编排。
  • public/js/logic.js：纯决策（禁 DOM/socket），与 node:test 共用。
  • public/js/canonicalize.js：前后端共享操作规范化（边界白名单）。
  • public/js/app/*：会话、附件、审批、设置、通知等子控制器。
  • public/vendor/：第三方库本地自托管，不依赖 CDN。
