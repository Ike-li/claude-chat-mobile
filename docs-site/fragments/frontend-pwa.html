前端是原生 ESM 的单页 PWA：Express 直接托管 public/，无打包步骤、无 CDN 运行时依赖。

分层

路径角色
public/index.html壳、资源引用、PWA meta
public/css/app.css样式（配合 Tailwind 运行时配置）
public/js/app.js编排：socket、UI 状态、子模块装配（大壳）
public/js/logic.js纯决策函数（禁 DOM / socket）；可被 node:test 直接测
public/js/canonicalize.js操作规范化；与后端共享白名单文件
public/js/ring-buffer.js客户端侧缓冲辅助
public/js/app/*会话、附件、审批、设置、通知、同步、渲染、任务状态等控制器
public/js/sw.js · sw-cleanup.jsService Worker 与清理
public/vendor/Tailwind / marked / highlight / DOMPurify / 字体 本地自托管
manifest.webmanifest · icons/可安装 PWA 与 maskable 图标
建议logic.js 红线：只做数据→数据。副作用（DOM、socket、localStorage 写）放在 app/* 控制器。单测优先覆盖 pure 函数。

连接与 token

  1. 首次通过 URL hash #token=… 注入鉴权密钥。
  2. 写入 localStorage 后从地址栏清除，避免分享 URL 泄露。
  3. 设备 token 同样持久化，参与 TOFU 握手。
  4. Socket 连接后进入事件分发（event-dispatch 等）；按 viewingInstanceId 分流。
  • 非当前 viewing 实例的高频 text_delta / thinking_delta 可能不直播到该 tab；切回后靠 sync:since 回放缓冲。
  • 重连后携带 lastSeq / epoch；epoch 变化时重置去重基线。
  • 发送忙碌锁（pendingSendBusy 等）与消息去重须按会话隔离；切会话后不得沿用上一会话的 busy / 重复 clientMessageId 判定。
  • server ack 受视图守卫：切 tab 后旧 send/cancelQueued 的回包不应驱动当前 composer 状态。
  • 限流重试中点「停止」：control_request 超时后应收口「正在停止…」，勿永久卡住（2026-07-20）。
  • 首页「最近活跃」含 worktree 会话，可冷启动一键续。

PWA 与推送前提

  • manifest.webmanifest + maskable 图标，支持「添加到主屏幕」独立显示。
  • iOS Web Push：需先安装到主屏幕且 https；局域网纯 http 场景 Web Push 不可用 → 配 ntfy。
  • Service Worker 负责推送相关与缓存策略（实现以 sw.js 为准）。

与边界守卫的关系

public/js 不得 import src/。允许共享的仅白名单：

  • public/js/canonicalize.js（后端 fingerprint 等使用）
  • public/js/logic.js（允许列表中；后端当前实际主要用 canonicalize，见漂移清单）

测试方式

E2E 使用 Playwright 移动视口 + 仓库内 mock server（零外部 token）。断言基于 DOM 状态，非像素比对。场景注册受 visual mock registry 门禁约束，mock 发出的 agent:event type 不得超出契约。

inventory 中前端源约 21 文件、vendor 约 10、图标为生成资产（scripts/gen-icons.js）。
