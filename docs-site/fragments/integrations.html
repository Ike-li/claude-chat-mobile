外部依赖只做「桥」与「通道」，不承接多租户账号，不替代本机 CLI。每一层都有明确的做与不做。

职责对照

集成做什么不做什么
Claude Agent SDK + 本机 claude CLI长驻 query()、流式输入、权限回调、spawn CLI；继承本机 settings / CLAUDE.md / MCP / skills / hooks不重新实现 Claude；不注入独立 allowedTools 清单；不自带登录态
Cloudflare Tunnel + Access（jose 验 JWT）固定域名把公网流量落到 localhost:3000；公网 Host 上强制 Access JWT（三项 env 齐全才启用）不替代 AUTH_TOKEN（LAN/本机仍靠 token）；不负责业务鉴权细粒度授权
web-push（VAPID）浏览器推送：审批、提问、结果等（需 https；iOS 先加主屏幕）未配 VAPID 则优雅缺席；不保证息屏必达
ntfy服务端 POST 通知，绕过 iOS PWA / 局域网 http 限制；可配深链 PUBLIC_URL未配则优雅缺席；公共裸 topic 不安全（正文可能含命令线索）
附注ANTHROPIC_* 只认启动 server 的 shell 环境；写进 .env 会在启动期剥除。官方订阅与第三方网关都走同一 CLI/SDK 路径，差别只在 shell 里 export 了什么。

运行时 npm 依赖

生产依赖（package.json）：

  • @anthropic-ai/claude-agent-sdk（探针时 0.3.201）
  • express · compression · socket.io · dotenv
  • web-push · jose

前端第三方不经 npm 运行时路径，而是自托管到 public/vendor/（Tailwind / marked / highlight.js / DOMPurify 等），见 public/vendor/THIRD-PARTY-NOTICES.md。

可选与缺席

  • CF_ACCESS_HOSTNAME / TEAM / AUD 任一缺失：Access 整层关闭，公网与 LAN 都回退 token 模型。
  • VAPID 三项不全：Web Push 关闭；仅 ntfy 或无推送仍可交互（需打开页面审批）。
  • statusline:install 不随 npm install 自动执行；CLI bridge 显式安装后才与终端 statusLine 同步。
  • 项目不内置 cloudflared 生命周期管理；凭据在 ~/.cloudflared，模板在 deploy/。
