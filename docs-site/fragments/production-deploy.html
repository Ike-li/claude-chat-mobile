生产形态是两个常驻进程加固定域名：Access 守公网，隧道落到本机 loopback，server 经登录 shell 启动以继承与终端一致的 PATH 与登录态。

架构

手机 → 域名 → Cloudflare Access（2FA）→ 命名隧道 → localhost:3000 → server
公网入口
固定域名；Access 验 OTP / IdP；公网路径不带 #token=
LAN / 本机
仍可用 http://<lan-ip>:3000/#token=…，走 AUTH_TOKEN
两个常驻进程
server（node）与 tunnel（cloudflared）；崩溃自拉起、关终端不掉

从零搭建（摘要）

  1. 隧道。cloudflared tunnel login → create → route dns → 写 ~/.cloudflared/config.yml ingress 到 http://localhost:3000。凭据勿提交。
  2. Access。Zero Trust 添加 Self-hosted 应用；取 AUD tag；填入 CF_ACCESS_HOSTNAME / TEAM / AUD 三项。
  3. 常驻。用 deploy/*.plist.template + scripts/render-plist.js 渲染后 launchctl bootstrap（Linux 换 systemd，思路相同）。
  4. 控制面外置。设 CCM_DATA_DIR 绝对路径 0700；doctor 再 kickstart。
  5. 可选通知。VAPID 与/或 ntfy + PUBLIC_URL。

LaunchAgent 模板

  • deploy/server.plist.template — 经 zsh -lc 'cd <repo> && exec <node> server.js' 启动，保 PATH/登录态；RunAtLoad + KeepAlive
  • deploy/tunnel.plist.template — cloudflared tunnel run <name>
  • deploy/log-rotate.plist.template — 定时 scripts/rotate-logs.sh（copy-truncate）
node scripts/render-plist.js deploy/server.plist.template \
  ~/Library/LaunchAgents/com.you.ccm-server.plist \
  LABEL=com.you.ccm-server REPO="$PWD" NODE="$(command -v node)" \
  LOG="$HOME/Library/Logs/ccm-server.log"

launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.you.ccm-server.plist
附注用 render-plist.js 做字面量替换与 XML 转义；路径含 & / # / 空格时勿用裸 sed。

mkdir -p "$HOME/Library/Application Support/claude-chat-mobile/data"
chmod 700 "$HOME/Library/Application Support/claude-chat-mobile" \
  "$HOME/Library/Application Support/claude-chat-mobile/data"

运维铁律

  • 生产端口已被常驻占用：勿再 npm start（撞端口）。
  • 改 .env 或代码后须 launchctl kickstart -k gui/$(id -u)/<label>（或 systemd restart）才生效。
  • 例外：WORK_DIRS_FILE 热加载免重启 server。
  • LaunchAgent 在登录后启动；开机未登录也跑需改 LaunchDaemon（进阶）。
危险反代到 loopback 会使设备审批按「本机」跳过。本配方公网靠 Access JWT 兜底；换成其它本机反代时务必强 AUTH_TOKEN，勿只靠设备层。详见安全模型。
