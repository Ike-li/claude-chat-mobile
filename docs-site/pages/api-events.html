事件与端点参考

入向 socket 事件、agent:event 类型、HTTP 运维端点。

阅读约 12 分钟 第 26 / 28 篇 源自 scripts/agent-event-contract.js · src/server/http.js

入向 socket 事件、出向 agent:event 类型，以及鉴权 HTTP 运维端点。权威清单是源码 scripts/agent-event-contract.js（探针时：入向 32、出向 type 25）。历史不走开放 HTTP。

入向 Socket（客户端 → 服务端）· 32

INBOUND_SOCKET_EVENTS 完整列表（按字母序与源码 freeze 数组一致）：

分组事件（共 32）
会话（9）session:close session:delete session:deletePermanent session:history session:home session:list session:new session:switch worktree:sessions
同步（3）conn:ping mirror:syncNow sync:since
用户控制（10）user:answer user:approve user:approveDevice user:cancelQueued user:denyDevice user:interrupt user:message user:setEffort user:setPermissionMode user:setViewing
任务 / 工具（3）task:stop tool:full tool:preview
文件浏览（2）browse:list browse:read
运维 / 诊断（5）dev:restart doctor:run logs:clientError logs:get service:status
附注历史经 session:history；断线补发用 sync:since。缓冲 cap 2000。user:setEffort 触发 dispose+resume 置换实例，不是 in-place 改活 SDK query。
建议新增入向事件时必须改 INBOUND_SOCKET_EVENTS 与 server 注册面，否则 npm run check / contract-check 失败。本页若与源码不一致，以源码为准。

出向 agent:event 类型 · 25

AGENT_EVENT_TYPES 完整列表：

分组type
生命周期（6）init instances result error system session_log
流式 / 历史（4）text_delta thinking_delta user_message history_append
工具（2）tool_use tool_result
权限 / 问答（4）permission_request permission_mode question request_resolved
模型 / 强度（2）models effort_mode
镜像 / 状态（3）mirror_state status_line api_retry
设备（2）device_status pending_devices
后台任务（2）task_notification task_progress

信封：{ seq, epoch, sessionId, instanceId, cwd, ts, type, payload }。其中 task_progress 等可经 emitTransient 发出（不占 seq、不进缓冲）。
