架构演进主线

同步引擎、单驾驶员、门禁、文档极简等关键阶段。

阅读约 12 分钟 第 6 / 28 篇 源自 git log · CLAUDE.md

今天的架构来自一串可观察的决策，而不是一次性设计文档。以下主线由 git 历史与现行代码交叉归纳。

主线阶段

桥接层

Web ↔ Agent SDK ↔ CLI

以 streaming input 与事件映射为中心；server 作为契约层而非业务 agent。

同步

信封 · 缓冲 · catch-up

seq/epoch 信封、2000 条环形缓冲、sync:since 重连补发；断线可恢复。

驾驶权

单驾驶员 + 只读镜像

Web 与终端不可同时写同一会话；CLI 驾驶时 Web 只读追平 transcript，发送路径接管。

安全纵深

token · TOFU · Access

无 token 绑 127.0.0.1；设备指纹审批；可选 Cloudflare Access JWT fail-closed。

工程门禁

check 流水线

ESLint、import 边界、双向事件契约、inventory、文档一致性；AI 会话下防止结构回缠。

文档极简

宣传层撤出主分支

2026-07-19 起设计长文与演示场景下线；产品 docs 只留部署与 inventory；宣传物迁 gh-pages / promo。

近期可观察提交主题（dev）

主题代表方向
同步 / mirrorexternalDirty、镜像锁自动释放、busy reseed、重连误判修复；2026-07-20：镜像误重锁、catchUpTickOnce 状态提交延后到视图守卫后
消息子系统pendingTurns 双扣修复、send/cancelQueued 视图守卫、pendingSendBusy 跨会话隔离、message-dedup TOCTOU；control_request 超时保护扩到 setModel/setPermissionMode/stopTask/cancelAsyncMessage
状态可见性CLI 式动态状态行、回合收尾行、服务状态面板判定化；限流重试中点停止不再卡「正在停止…」
后台 / 子代理多子代理并行时防 idle 误杀与误解锁；bgTasks 刷新 lastActivity；生命周期文案理顺
worktree会话触达接线；首页「最近活跃」纳入 worktree 会话（冷启动一键续）
排队与交互排队可见、撤回重编辑、原生 confirm 改 sheet
运维日志时间戳、copy-truncate 轮转、客户端错误上报
仓库边界宣传层下线、demo 场景拆除、.worktrees 约定

手册探针基线：dev@2de707b（自 8084005 起含上述消息子系统审查修复与 worktree 首页）。

有意保留的「上帝模块」

src/server/app.js 仍是组装根本体：mirror / catchUp / openInstance 与契约路由共享同一组顶层可变状态（viewing*、mirror*、catchUp*）。域逻辑已下沉，但接线有意不拆成多个上下文穿针——注释与边界守卫共同说明这一点。

附注代码注释中大量 docs/design.md / AD-x / NFR-x 溯源标记是历史决策锚点；文件本身已删除，标记有意保留，勿当作仍存在的文档路径。

分支与产物解耦

  • 产品开发线：master / dev。
  • 展示：gh-pages 孤儿分支（landing + 本手册）。
  • 文案：promo 孤儿分支。

宣传物与产品源码解耦，避免主分支再次被素材与生成脚本淹没。
