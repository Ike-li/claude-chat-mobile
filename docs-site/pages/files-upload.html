文件与上传

工作区白名单、三层路径闸、附件与预览。

阅读约 10 分钟 第 17 / 28 篇 源自 src/files/

文件能力围绕「白名单工作区」与「只读不越界」：上传附件进会话，浏览与预览不得逃出允许目录。

工作区白名单

  • 默认 cwd：WORK_DIR（未设则 home）。
  • 附加目录：WORK_DIRS（逗号串）或 WORK_DIRS_FILE（JSON 文件，优先）。
  • WORK_DIRS_FILE 支持热加载：server 监听文件变更（父目录 watch + mtime + debounce）；改完免重启。
  • 被移除目录上已有 live 实例可继续；该目录上新开会话拒绝（ensureWhitelisted）。
  • 无效目录 soft-skip 并告警，不挡启动。
以代码为准口语常说「改 workdirs.json」。契约环境变量名是 WORK_DIRS_FILE（值通常指向仓库根 workdirs.json）。未设置该 env 时不会 watch，改文件也不热加载。

附件上传流程

  1. 客户端经 socket 上传字节（校验类型与大小，见 validateAttachments）。
  2. 服务端写入 <当前 cwd>/.ccm-uploads/<ts>-<rand>-<safeName>（常量 UPLOAD_DIR = '.ccm-uploads'）。
  3. buildPromptText 把绝对路径注入 prompt，供 CLI / 工具 Read。
  4. 事件 meta 带缩略信息与 storedName；历史消息可按需再拉预览（browse 读盘，非无鉴权 HTTP 图片端点）。
留意transcript 记录的是当时绝对路径。挪工作区、改机器用户名或清空 .ccm-uploads 后，旧附件可能无法打开。备份对话若仍要看图，须一并备份对应上传目录。

路径闸（三层）

层模块作用
写安全file-security.jsowner-only 写（0600）、符号链接危险组件拒绝、原子写辅助
范围门workdir-scope-guard.js路径必须落在白名单 realpath 之下
浏览/预览file-browse.js · file-preview.js只读列表、片段与 diff；工具卡片同源约束

工具卡片对 Edit / Write 显示 diff、对 Read 显示片段，均限白名单工作目录内，只读不越界。

相关 socket 事件

  • 上传随 user:message 附件字段进入主发送路径。
  • browse:list / browse:read：工作区内只读浏览（含历史附件按需读）。
  • tool:preview / tool:full：工具结果预览与全文拉取。

实现入口：src/files/*、src/server/socket-files.js、src/sessions/workdirs.js。
