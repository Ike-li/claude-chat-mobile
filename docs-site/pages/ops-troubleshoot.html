运维与排错

重启、日志轮转、/health /metrics、常见故障。

阅读约 12 分钟 第 24 / 28 篇 源自 docs/deployment.md · scripts/doctor.js

常驻实例的日常操作、排错表、探针端点与 doctor 摘要。改 env 不生效，十有八九是忘了 kickstart。

运维速查

# 日志（轮转归档 <name>.0.gz … .4.gz）
tail -f ~/Library/Logs/<your-server-log>.log

# 重启 / 停 / 起（macOS LaunchAgent）
launchctl kickstart -k gui/$(id -u)/<your-server-label>
launchctl bootout    gui/$(id -u)/<your-server-label>
launchctl bootstrap  gui/$(id -u) ~/Library/LaunchAgents/<your-server-label>.plist

# 是否在跑（有 PID = 在跑）
launchctl list <your-server-label> | grep -E '"PID"|LastExitStatus'

# 临时本地调试：先让出端口
launchctl bootout gui/$(id -u)/<your-server-label>
npm start

排错表

现象处理方向
公网 502 / 1033server 未跑：看 server 日志并 kickstart；或隧道挂了：看 tunnel 日志是否有 Registered tunnel connection
OTP 过了但 app 连不上JWT 校验失败：日志搜「Access JWT」；核对 CF_ACCESS_TEAM / AUD 与 CF 应用一致
手机进不了登录页DNS / 隧道连通；ingress 指向 http://localhost:3000
改 env 不生效常驻进程未重启；kickstart / systemctl restart
model_not_found模型名可能需后缀；在启动 shell export ANTHROPIC_MODEL 后重启，或 web 端 /model 切换（.env 中 ANTHROPIC_* 被剥）
部署机全局代理仍 1033代理 TUN 劫持 cloudflared 到 edge；需给 edge 直连路由或绕过 TUN
只有工具卡片无正文部分网关不流式；agent map 有全文兜底；仍复现则 LOG_STDERR=1 看子进程
手机连不上且无 AUTH_TOKEN只绑 127.0.0.1——有意设计；设 token 并重启
设备一直 pendingnode scripts/device.js list / approve；或检查是否被反代当成「本机」而路径异常

探针与鉴权

端点说明
GET /health健康 JSON；经 httpAuth
GET /metrics指标快照；同样鉴权，不落盘、非 Prometheus 文本
GET /push/vapid-public-key · POST /push/subscribeWeb Push；鉴权

业务历史与会话数据走 Socket（如 session:history），不开放无鉴权的 HTTP 数据端点。限速与握手共用，防无限试 token。

日志轮转

scripts/rotate-logs.sh 使用 copy-truncate：launchd 持有 O_APPEND fd，rename 式轮转会把新文件写成空。默认超约 20MB 才转，gzip 保留 5 份。由 log-rotate.plist.template 定时触发。

诊断开关DEBUG_SDK_MESSAGES / LOG_INTERACTIONS / LOG_STDERR 会显著放大日志；生产确认问题后关掉并重启。面板「服务状态」可看到部分开关是否打开。

doctor：两套面，勿混为一谈

CLI 自检（scripts/doctor.js · 固定 11 项）

命令行 node scripts/doctor.js 跑 D1–D11，启动前配置诊断用：

#检查
D1AUTH_TOKEN 非空与长度
D2CLAUDE_BIN / which claude 可执行
D3WORK_DIR / WORK_DIRS(_FILE) 可写
D4PORT 未被占用
D5WEB_STATUSLINE 配置口径
D6CLI statusline bridge 安装态（只读）
D7网关环境：.env 中 ANTHROPIC_* 会被剥除的提示
D8.env / data JSON 权限（owner-only）
D9文档一致性（死链、旧文件名、scripts、SDK 版本）
D10前端 public/js/**/*.js 语法
D11单测行覆盖率门槛（≥65% 未过则 warn）
 
node scripts/doctor.js
node scripts/doctor.js --env=prod.env
node scripts/doctor.js --fix

UI 体检（运行中 · doctor-runtime / doctor-checks）

手机/Web 面板触发的 doctor:run 走 src/ops/doctor-runtime.js，与 CLI 11 项不是同一清单。UI 侧会给出脱敏安全视图，额外包含例如：

  • CF Access 是否启用、AUD 是否配置（仅布尔）
  • 设备门 / 反代 loopback 是否跳过 TOFU（DEVICE_GATE）
  • Web Push VAPID 是否齐全
  • 信任/待批设备计数
  • CLI settings 危险白名单摘要（WHITELIST）

排障时：启动前问题优先 CLI doctor；线上「现在这台实例安全吗」看 UI 体检与服务状态面板。
