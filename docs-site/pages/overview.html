01 第一部分 · 认识项目

项目总览

定位、适用场景、技术栈与仓库形态。

阅读约 10 分钟 第 3 / 28 篇 源自 README.md · CLAUDE.md

项目给已经在终端使用 claude CLI 的人用：在手机上对同一 agent 打字，效果与坐在电脑前等价。

一句话定位

移动端聊天式 Web UI，通过 Claude Agent SDK 驱动本机已登录的 claude CLI。不打包 Claude，不重实现 agent 循环；继承同一份 CLAUDE.md、MCP、skills、hooks 与会话 transcript。

适用场景

  • 任务在跑，人离开电脑。审批与提问推到手机，点允许或拒绝。
  • 同一会话，换设备继续。手机与终端读同一份 CLI 会话记录；路上起头，回家 /resume 接上。
  • 多仓库并行。白名单内切换工作目录，标签页看多个会话进度。
  • 手机输入更顺。斜杠命令、相册附图、长按复制等按触控设计，而非缩小终端。

若只是偶尔远程看一眼屏幕，远程桌面通常够用。本项目面向「把手机当成终端随身入口」的高频使用。

技术栈

运行时
Node ≥20 · ESM · Express · Socket.IO 4 · compression · dotenv
Agent
@anthropic-ai/claude-agent-sdk 0.3.201 · 本机 claude
鉴权 / 推送
jose 6（Access JWT）· web-push（VAPID）· 可选 ntfy
前端
原生 ESM PWA；vendor 自托管（Tailwind / marked / highlight.js / DOMPurify）
测试
node --test 单测/集成 · Playwright 移动端 E2E · smoke 真 CLI（默认不跑）
许可
AGPL-3.0-only · NOTICE Section 7 补充条款

仓库形态

路径角色
server.js兼容启动器（npm / LaunchAgent / systemd）
src/服务端实现，按域分包
public/Express 直接托管的 PWA
scripts/维护命令，非运行时
tests/唯一自动化测试根
docs/极简：deployment + repository-map
deploy/plist 模板

inventory 分类文件约 277 个（见生成的 docs/repository-map.md）。后端源约 44 文件，前端源约 21，单测约 89，E2E 约 30。

分支纪律

  • dev：日常开发。
  • master：稳定 / 默认克隆目标；有分支保护；经 scripts/release.sh 从 dev ff 合并并发版。
  • gh-pages：宣传站孤儿分支（本手册产物所在）。
  • promo：宣传文案创作区，非产品源码。
附注仓内 .worktrees/ 是其他分支的常驻检出位，不是 dev 源码。在 dev 上搜索/审查应排除该目录。

前置条件

  1. Node ≥20。node --version 检查。
  2. 本机可用的 claude CLI。which claude 且能登录对话；项目不自带 CLI。
  3. 订阅或网关。官方订阅沿用 CLI 登录态；第三方网关在启动 server 的 shell 中 export ANTHROPIC_*（写入 .env 会被剥除）。
  4. macOS 或 Linux。
