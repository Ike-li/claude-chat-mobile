快速开始

安装、setup、doctor、局域网与临时隧道。

阅读约 10 分钟 第 22 / 28 篇 源自 README.md · scripts/setup.js

最短可跑路径与 README 一致：装依赖、写 .env、doctor、起服务。同 WiFi 必填 AUTH_TOKEN；公网用临时隧道即可试用。

前置

  • Node ≥ 20（node --version）
  • 本机已安装并登录的 claude CLI（which claude）
  • macOS 或 Linux
  1. 克隆并确认前置。 
    git clone https://github.com/Ike-li/claude-chat-mobile.git
cd claude-chat-mobile
node --version
which claude
  2. 安装运行依赖并交互配置。 
    npm install --omit=dev   # 仅运行依赖；跑测试用完整 npm install
npm run setup            # 生成 AUTH_TOKEN、询问 WORK_DIR，写入 .env（0600）
# 或：cp .env.example .env 后手改
  3. 自检。 
    node scripts/doctor.js
node scripts/doctor.js --fix  # 收紧 .env 与 data JSON → 0600
  4. 启动。 
    npm start                 # http://localhost:3000
# 启动日志会打印带 token 的可用 URL

如何从手机连上

场景做法
同一 WiFi在 .env 设非空 AUTH_TOKEN，打开启动日志中的 http://<lan-ip>:3000/#token=…。无 token 时只绑 127.0.0.1，手机连不上（有意为之）。
临时公网 / PWA https另开终端：cloudflared tunnel --url http://localhost:3000，手机打开随机域名并带 #token=。token 首次写入 localStorage 后从地址栏清除。
固定生产命名隧道 + Access 2FA + 常驻进程；见生产部署。
留意ANTHROPIC_* 只认启动 server 的 shell。第三方网关请在该 shell 中 export 后再启动。写进 .env 会被剥除。官方订阅则依赖 CLI 自身登录态即可。
建议可选 CLI 镜像状态栏：npm run statusline:install（不会随 install/start 自动装）。查看：npm run statusline:status。多数用户只开 Web 驾驶时不需要。

开发模式

npm run dev    # node --watch server.js
npm run check  # 推送前零 token 门禁

生产常驻已占 3000 时，勿再手动 npm start 撞端口；临时调试先 bootout 常驻服务。更多见运维与排错。
