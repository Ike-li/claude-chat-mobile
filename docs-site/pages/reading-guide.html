阅读顺序指南

主线通读、30 分钟速览、按角色切入。

阅读约 8 分钟 第 2 / 28 篇

读者打开手册后应立刻知道：怎么读、读哪条线、按什么顺序。主线兜底，速览与角色线是捷径。

主线通读

主线 = 七部分的天然顺序，与侧栏一致：

起点 → 认识 → 演进 → 方法论 → 实现 → 数据 → 运维 → 规范

预计 2–3 小时。适合第一次完整接手。每页可独立读懂，但仍建议按序建立心智模型。

30 分钟速览

  1. 封面与总览。封面 → 项目总览：一句话定位与场景。
  2. 架构与边界。系统架构 → 职责边界：分层与「不做什么」。
  3. 方法论三页。终端等价 → 安全模型 → 单驾驶员。
  4. 能跑起来。快速开始；若要公网再扫生产部署标题。
  5. 校准事实。漂移清单：文档与代码不一致处以代码为准。

按角色切入

角色先读再补
自托管运维快速开始、生产部署、运维排错、环境变量安全模型、鉴权与设备
后端 / 同步架构、AgentSession、事件信封、单驾驶员、数据全景会话与历史、契约门禁
前端 / PWA前端与 PWA、事件信封、API 事件参考通知与状态栏
安全审查安全模型、鉴权与设备、文件上传、职责边界生产部署中的反代警告
贡献者项目规范、契约门禁、术语表演进主线、漂移清单

事实基线约定

以代码为准手册基于 dev 源码交叉核对（探针 HEAD 2de707b），不是持续自动同步。注释里若仍出现 docs/design.md，该文件已从仓库移除；权威安全叙述在 README「安全模型」节。矛盾点集中记在漂移清单。
搜索任意页按 ⌘K / Ctrl+K 打开全文搜索。深浅色由顶栏切换，偏好写入 localStorage。
