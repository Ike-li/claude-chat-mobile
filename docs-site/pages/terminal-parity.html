终端等价性

CLI 有什么 Web 就有什么；权限与配置同源。

阅读约 10 分钟 第 7 / 28 篇 源自 CLAUDE.md · README.md

Claude Chat Mobile 的产品目标是终端等价：CLI 有什么，Web 就有什么。实现上不重造 Agent 能力，而是用 Agent SDK 驱动本机已登录的 claude CLI，并与 CLI 共用配置、权限与 transcript。

等价原则

当你不知道怎么处理功能时，CLI 有什么 web 就有什么，请找一找 claude code cli 是怎么实现这个功能的。—— CLAUDE.md

本项目不是远程桌面，也不是另一套 Claude 客户端。手机上的一次发送、一次审批、一次切模型，应与坐在本机终端前操作同一会话等价。遇到「Web 还缺什么」时，默认动作是找 CLI 现有实现与配置面，而不是在转发层另开一套语义。

01

同源驱动

Agent SDK 拉起本机 CLI 子进程，而不是另写一套工具循环。

02

同源配置

settingSources: ['user','project','local']，与 CLI 读同一组 settings。

03

同源会话

transcript 落在 ~/.claude/projects，与 CLI /resume 同一事实源。

配置加载：settingSources

settingSources: ['user', 'project', 'local']

含义与 CLI 一致：用户级、项目级、本地覆盖依次参与合并。模型默认、权限白名单、env 块等由这些 settings 决定，转发层不维护平行配置权威。

建议改权限或默认模型时，先改 ~/.claude/settings.json 或工作区 .claude/settings.json[.local]，再在 Web 验证；不要在 CCM 里另造白名单文件。

权限：不注入 allowedTools

SDK 选项不注入 options.allowedTools。放行白名单完全交给 settingSources 加载的 settings 里 permissions.allow 并集（与终端 claude 同源、由用户自管）。SDK 把 settings 的 allow 当作「自动放行、不触发 canUseTool」的第一层；未命中则进入统一闸门 canUseTool。

层行为
settings permissions.allow命中则自动放行，不弹手机审批
canUseTool白名单外统一闸门；按当前权限档 allow / deny / 弹窗
转发层自建白名单不维护；避免与 CLI 分叉
留意公网暴露前务必审查全局 ~/.claude/settings.json。终端里多年累积的 Bash(…) / Write 规则在手机侧也会自动放行，不会再弹审批。

权限档

代码白名单（setPermissionMode / CCM_PERMISSION_MODES）为 六档：

档含义
default默认审批；白名单外走 canUseTool 弹窗
plan计划模式，与 CLI plan 语义一致
acceptEdits接受编辑类操作的放宽档
bypassPermissions实例记真实 bypass；给 SDK 映射为 default，由 handleCanUseTool 直接 allow（避免 SDK 危险全局 flag）
dontAskWeb 专属严格档：白名单外 deny、手机不弹窗；CLI transcript 不会出现
autoSDK 用模型分类器自动批准/拒绝权限请求

从 CLI transcript 恢复时，可读档为 default / plan / acceptEdits / bypassPermissions 等 CLI 可写出的档；dontAsk 属 Web 专属，不会从 CLI transcript 读回。

会话与 transcript

~/.claude/projects/<编码 cwd>/<session_id>.jsonl

CCM 的 sessions.json 只存元数据，不存消息正文。列表与历史冷读与 CLI /resume 同源扫描该目录；历史封顶 HISTORY_MAX_MESSAGES = 2000。

模型与 effort

模型可在发送路径切换。思考强度 effort 在 spawn 时注入 SDK options（与终端 /effort 同旋钮）；SDK 无运行时 effort 控制请求。Web 侧 user:setEffort 的切档路径是 server dispose 当前实例 + 同 sessionId resume 新实例（instanceId / epoch 会变），不是 in-place 改活进程。新会话（FRESH）的默认档来自 CLI settings 合并结果，而不是「继承本 cwd 上一个 Web 实例的末档」——贴近「新起一条 claude 是干净默认」。

发送与接收路径

发送：Web → Agent SDK → Claude Code CLI
接收：Claude Code CLI → Agent SDK → Web

刻意不对齐之处

  • 手机 UI 是触控优先的聊天壳，不是终端仿真器（斜杠命令、相册附图、长按复制等）。
  • Web resume 是冷读 transcript，不能 attach 正在运行的 CLI 活进程（见单驾驶员）。
  • 通知、PWA、设备 TOFU 是桥接层能力，CLI 本身没有对应物。
  • 状态文案 / spinner / statusline 尽量对齐 CLI 语义，交互壳可以是手机 UI，但权限与会话语义不得分叉。
