# 在手机上，你能用 claude 做什么

**中文** · [English](capabilities.en.md)

> 这份文档回答一个问题：**装好之后，手机能干什么。** 配置步骤见 [README](../README.md)，安全模型见 [design.md §4](design.md)。

先说清一件事：本项目**不是一个新的 AI，也不是 Claude 的重新实现**。它把你**本机已经在用的 `claude` CLI**（Claude Code）投送到手机浏览器——所以你在终端里拥有的 agent 能力，手机上基本原样都在：同一个 agent、同一份 `CLAUDE.md`、同样的 MCP 服务器、技能、hooks 和已登录会话。设计目标是**终端等价性**：手机上对 claude 打字，效果和坐在电脑前打字一样。

下面按「你在手机上做的事」来讲，每一块都标出 **↳ 背后间接用到的 claude CLI 能力**。

---

## 一、像在终端一样，和 claude 对话、真的干活

- **发消息、看它逐字流式回复**——不是等一整段，而是像终端里一样实时长出来。
- **它真的改文件、跑命令**，不是只聊天：`Read` / `Edit` / `Write` / `Bash` / `Grep` 等工具都会真实执行在你的项目目录里。
- **全过程可见**：每一次工具调用都渲染成一张可折叠卡片（工具名 + 参数摘要 + 执行状态），你能看清它此刻在干什么，而不是一个黑盒。
- **Markdown 与代码高亮**：小屏适配，代码块可横向滚动，长回复也读得下去。
- **随时喊停**：一个停止按钮中断当前生成，会话状态不受损，之后还能接着来。

> ↳ **背后**：同一个 claude agent 与它的真实工具执行；流式输出、工具调用、中断都直接来自 Claude Agent SDK，中间层不删改、不摘要、不"优化"它的输出。

---

## 二、手机端的核心价值：危险操作回手机审批

这是本项目和「远程桌面看一眼」最不一样的地方。

- claude 要做**白名单以外**的事（比如 `git push`、`rm`、改配置、装依赖），会**挂起并把审批请求推到你手机**——弹窗里是**完整命令 + 工作目录 + 关键参数**，不是只给个工具名。
- 你点 **允许 / 拒绝 / 本会话总是允许**，claude 才继续。
- **五种权限档运行时可切**：`default`（默认逐个问）、`plan`（只规划不动手）、`acceptEdits`（自动接受改文件）、`bypassPermissions`（全放行，爆炸半径=整机，需显式切+危险确认）、`dontAsk`。

> ↳ **背后**：自动放行的范围**完全等于你已有 claude 配置里的 `permissions.allow`**（全局 `~/.claude/settings.json` + 项目 `.claude/settings.json` + 本地 `.claude/settings.local.json` 合并，与终端同源）——本项目**不注入自己的任何放行/禁止清单**。命中即放行，未命中就走 SDK 审批回调、弹到你手机。

---

## 三、你的整套 Claude 环境，原样继承

手机端操作的是**"你的 Claude Code"**，不是一个干净的默认实例。

- **你的 `CLAUDE.md`、MCP 服务器、skills、hooks、已登录会话**——全部加载，行为和你终端里一致。
- **斜杠命令**：输入 `/` 弹出**你实际拥有的**命令列表（`/skill`、你的自定义命令等），点一下就执行。
- **逐条消息切模型**（`/model`，支持网关后缀名）、**切换思考强度**（thinking effort）。
- **claude 反问你时**（`AskUserQuestion`），手机上呈现为**原生选项按钮**，点一下作答，而不是让你手打编号。

> ↳ **背后**：通过 SDK 的 `settingSources` 加载你本机的完整配置；斜杠命令来自会话 `init` 事件的 `slash_commands`（claude 机器可读的命令边界）；模型与 effort 原样透传给 CLI。

---

## 四、为"手机使用模式"而生（这些终端没有）

手机的使用方式是「看 30 秒 → 锁屏 → 十分钟后再看」，本项目专门为此设计：

- **会话连续、跨设备接力**：手机上起个头，回电脑 `/resume` 无缝接上——两端读的是**同一份 CLI 会话记录**，不是各开一摊。
- **离开也不断**：任务挂在服务端、和网络连接解耦。锁屏、切后台、地铁断网，它都在跑；回来页面**自动重连并补齐你错过的全部输出**。
- **弱网自愈**：每个事件带单调序号，重连按序号续传，而不是丢失或重复。
- **历史会话**：会话列表里选择、切换、新建，切进去**回显完整历史消息**（与终端 `/resume` 同源）。
- **多仓库、多会话并行**：在白名单允许的工作目录之间切换，用标签页同时跑多个会话——一块屏的远程桌面做不到。
- **上传文件 / 图片**：从相册或文件选一个丢给 claude，它当场 `Read`（带路径注入与穿越防护）。
- **复制长输出**：一个复制按钮拷出原始 Markdown 源文。
- **Web Push 通知**：**审批请求 / claude 的提问 / 你断线期间跑完的结果**会推到手机通知，不用一直盯着屏幕（iOS 16.4+ 需先"添加到主屏幕"）。
- **Web 自有状态栏**：输入框上方显示模型、上下文用量、成本、git 等状态（app 原生渲染，不复刻终端 ANSI）。

> ↳ **背后**：claude 原生 session（JSONL 会话文件）是唯一事实源，resume 全靠它；本项目只在服务端存一份轻量的会话元数据映射。手机端还会记住某会话上次生效的权限档 / 思考强度（这是相对终端的一点增强）。

---

## 手机操作 ↔ 终端等价对照

| 你在手机上… | 等价于坐在终端前… |
|---|---|
| 发一条消息 | 对着 claude 打字 |
| 看流式回复 + 工具卡片 | 看它逐字输出、看它调用工具 |
| 点审批弹窗的「允许 / 拒绝」 | 回答权限确认的 `y / n` |
| 点停止按钮 | 按 `Esc` 中断 |
| 敲 `/` 选命令 | 用 `/skill`、`/model`、自定义命令 |
| 点 `AskUserQuestion` 的选项 | 回答 claude 的反问 |
| 切换会话、回看历史 | `claude -r` 选历史会话、`--continue` 接着聊 |
| 锁屏十分钟后回来自动补输出 | 一直坐在那儿看着它跑完 |
| 收到审批 / 结果的手机推送 | 人就在终端前，出现即被看到 |

## 边界：哪些**不**做、哪些**看不到**（如实说明）

为了不误导，这些是有意为之或架构使然的限制：

- **不复刻终端的设置面板**：`/config`、`/plugin` 这类终端客户端**自己的 TUI 面板**不搬过来——它们用 app 自己的 UI 或自然语言替代。等价的对象是 **agent 能力**（对话、工具、技能、审批、resume、模型切换），不是终端客户端的界面。
- **续接一个终端里正在跑的活跃会话时，看不到实时状态**：手机续接是另起一个进程冷读磁盘上的会话记录，只能**只读追平已经落盘的输出**；一个 turn 进行中的子 agent、thinking 等尚未落盘的内容看不到，也不能从手机**接管**一个终端活进程的输入。这是架构限制，不是 bug。
- **不做会话重命名 / 删除、消息搜索 / 导出 / 统计面板**：终端也没有这些；真要找，`~/.claude/projects/` 里原始记录都在。
- **状态栏不含账号级配额**（如 5 小时 / 7 天用量百分比）——SDK 物理上拿不到这个数。
- **它不是远程桌面**：远程桌面镜像一块屏幕；本项目给的是你本机 claude 会话的**原生手机入口**——为手机而生的输入、通知、多会话，而不是把电脑屏幕投过来。
