Why it's different

A transparent pipe to your terminal — locked by default

01

Your real CLI, not a clone

It ships no Claude of its own. It drives your local CLI through the Claude Agent SDK, so you inherit the same agent, the same CLAUDE.md, the same MCP servers, skills, hooks, and logged-in session you use at your desk.

02

Terminal equivalence

Edit code, run commands, resume an earlier conversation — typing on your phone behaves exactly like typing at your computer. Except now you can do it from bed.

03

Approve danger from your phone

Off-allowlist actions are suspended and pushed to your phone with the full command and working directory. One tap to allow or deny — nothing runs until you say so.

04

Survives a flaky subway

The task lives server-side, decoupled from the socket. On a dropped mobile connection it self-heals and replays, so you land back where you were instead of reconnecting and re-locating.

Security model

A code-execution channel into your shell. Treated like one.

1

Single-user per instance

Open for anyone to self-host — you run your own instance, for yourself. Each instance is single-user by design: no accounts, no login. Anyone who passes auth has the same power as you at the terminal, so only ever point it at your own machine.

2

No token, no exit

With no AUTH_TOKEN set, the server binds to 127.0.0.1 only — there is no "empty = open to the world" path. Reaching the internet requires a token.

3

Pure inheritance, zero injection

The auto-approve set is exactly your existing CLI allowlist — nothing added. Anything off-list is suspended and bounced to your phone for approval.

4

Device trust (TOFU)

A connection that is neither local nor Access-verified must be authorized once on your computer before it can do anything. A valid token alone is not enough.

No AUTH_TOKEN ⇒ binds 127.0.0.1 only. It cannot reach the public internet until you deliberately open it.

Honest scope

When it's worth it

This isn't "open a remote desktop back to your computer." That mirrors a screen. This gives your real local claude session a native entry point built for a phone: one tap to approve, one session picked up across devices, several repos watched in parallel tabs, native photo upload and long-press copy.

If you only glance in remotely now and then, an existing remote desktop is enough. The value lands when you use your phone frequently as a pocket extension of your terminal.

Run it on your own machine

It's self-hosted by design — there is no shared demo, because it drives your own authenticated claude CLI.

Prerequisites: Node.js ≥ 20 on macOS or Linux, and a working, logged-in claude CLI on the host (this project drives it, it doesn't bundle it).

# clone, install runtime deps, configure, run git clone https://github.com/Ike-li/claude-chat-mobile.git cd claude-chat-mobile npm install --omit=dev cp .env.example .env # set AUTH_TOKEN, WORK_DIR npm start # http://localhost:3000
