# 全量人工手测清单（生产实例）

> 一次完整的人工手动回归。测试**在生产实例上跑**（公网域名 + Cloudflare Access 2FA + 常驻 server），因此会碰真实会话、消耗真实 token、触发真实设备/推送/审批——**先做完 Suite 0 的防护栏再动手**。
>
> 本清单挂靠 `docs/design.md` 的权威剧本（§2 E1~E16 终端等价性、§5 A1~A15 验收），不重造。自动化已覆盖的部分标注出来，人工只做抽查确认；只有人能测的做深测。

## 怎么用

- 按 **Suite 0 → 11** 顺序执行；每条一个可勾选块。
- 覆盖标签：`[纯人工]` 自动化够不到、必须深测 · `[集成/单测/视觉 已覆盖·抽查]` 有自动化守着、人工抽查即可 · `[smoke 可跑]` 有现成脚本可先跑一遍。
- 失败就在 `[ ] 失败` 后写现象 + 复现 + 锚点编号，最后汇总到 Suite 11。
- 占位符：`<your-domain>` 你的公网域名 · `<label>` 常驻 server 的 launchd/systemd 标签 · `<ID>` 设备 ID · `/Users/you/...` 你的路径。
