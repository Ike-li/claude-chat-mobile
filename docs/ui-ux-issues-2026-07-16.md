## P0——影响核心流程正确完成与基本可读

<a id="ux-001"></a>
### UX-001 · ExitPlanMode 计划内容以 JSON 转义串展示 · ✅ 已修

- **问题**：审批 sheet 的 `#permInput` 直接展示原始序列化串。ExitPlanMode 时计划书显示为 `"## 计划\n1. 实现 X\n2. 测试 Y"`（`\n` 字面量），审批人在手机上读的是密文而非格式化计划——而计划审批是 plan 工作流的关键决策点。普通命令审批同样带 JSON 引号（`"git push origin dev"`）。
- **证据**：实测截图（`test:exitplan` / `demo:permission` 场景，390×844）。
- **建议**：ExitPlanMode 的 input 用现成的 `renderMarkdown`（marked + DOMPurify 均已在依赖里）渲染进 sheet；普通命令去掉引号转义、保留 mono 展示。
- **涉及**：`public/index.html`（#permInput）、`public/js/app.js` 审批弹窗填充逻辑。

<a id="ux-002"></a>
### UX-002 · 工具卡收起态无操作对象 · ✅ 已修

- **问题**：工具卡收起态只显示工具名（`run_command` / `read_file`），不显示 `inputSummary`（命令内容 / 文件路径）。一轮多工具时出现连续多张同名卡片，无法区分；CLI 里 `Read(app.js)` 一眼可扫的信息，这里必须逐张展开。
- **证据**：实测截图（`demo:tool` / `demo:permission` 场景）；事件 payload 中 `inputSummary` 字段已存在且有值。
- **建议**：收起态标题改为「工具名 · inputSummary 截断」一行（如 `read_file · public/js/app.js`），truncate 溢出。
- **涉及**：`public/js/app.js` 工具卡渲染。

<a id="ui-001"></a>
### UI-001 · 四组核心文字对比度低于 WCAG AA（4.5:1） · ✅ 已修

- **问题**（程序化计算值）：
  | 组合 | 浅色 | 深色 |
  |---|---|---|
  | 白字 / accent 底（「允许」「进入」等主 CTA，14–15px） | **3.90 ❌** | **3.12 ❌** |
  | accent 文字 / surface（忙碌条"Claude 正在思考中…"） | **3.73 ❌** | 5.51 ✅ |
  | warning 文字 / surface（后台任务横幅 12px） | **3.14 ❌** | 5.25 ✅ |
  | danger / info 文字 / surface（镜像横幅、提问标题） | 5.04 / 5.06 ✅ | **3.27 / 3.25 ❌** |
  主 CTA 是 14–15px 中字号，不享受大字（3.0）豁免。
- **建议**：浅色主题 accent 底按钮改用 `--accent-deep`（#AE5238，白字对比 5.17）或文字加粗放大；深色 danger/info 单独提亮一档（覆盖值写进 dark `:root`）；warning 文字场景改深一档变体。
- **涉及**：`public/css/app.css` design token、按钮类。

<a id="ux-003"></a>
### UX-003 · 审批「允许」无防误触 · ✅ 已修

- **问题**：审批 sheet 从底部弹出，「允许」是 accent 实底主按钮、正落拇指热区，弹出瞬间即可点。手指若正悬在输入区附近极易误触授权。终端上"回车=允许"有键盘缓冲这道物理闸，手机没有——终端等价不应等价到误触风险。
- **证据**：实测截图（`demo:permission`，sheet 布局与按钮位置）。
- **建议**：sheet 弹出后 300–400ms 内按钮不可点（transition 期间 `pointer-events:none`）；可选：仅对 `run_command` 类高危工具把「允许」做成二段确认。
- **涉及**：`public/js/app.js` 审批弹窗打开逻辑、`public/css/app.css` sheet 过渡。

## P1——显著提升体验质感

<a id="ux-004"></a>
### UX-004 · 流式期间显示裸 markdown 源码

- **问题**：流式过程中正文以未渲染的 markdown 源码显示（`**手机 ↔ 本机 claude CLI**` 星号裸奔、列表序号未成形），回合结束才整体重渲——长回复边流边读被符号干扰，完成瞬间版式跳变。
- **证据**：实测截图（`demo:stream` 场景流式早期 vs 完成态对比）。
- **建议**：增量渲染——已完结段落走 `renderMarkdown`、尾部未闭合缓冲区原样追加；或每 delta 全量重渲当前消息（本产品消息体量下性能足够）。
- **涉及**：`public/js/app.js` text_delta 处理。

<a id="ux-005"></a>
### UX-005 · 侧边栏第一屏说明书化 · ✅ 已修

- **问题**：抽屉打开后第一屏依次是：4 项 emoji 图例、3 行角标语义解释（"目录角标 = 该工作区最需关注的状态"等）、「状态 · live 会话实例」7 项计数（live/运行/待批/出错/完成/中止/空闲）、实例列表、免责声明（"此处为 web 驱动的会话实例……无法检测 OS 僵尸"——开发者内部语言直接暴露给用户），真正要找的工作区/会话树被推到下半屏。需要常驻图例本身就是状态编码不自明的自白。
- **证据**：实测截图（默认态与 `test:needsyou` 态侧边栏）。
- **建议**：会话/工作区树上移第一屏；图例收进一个 (?) 弹出；计数缩成一行「live N · 需处理 M」；免责声明移入帮助页；"OS 僵尸/web 驱动实例"换成用户语言（如"仅显示由本应用发起的会话"）。
- **涉及**：`public/js/app.js`（sessionPanel 渲染）。
- **落地**：整段 live 实例汇总 + 状态图例已删；抽屉顺序 = 需要你 → 服务异常 → 工作区树。`whatNeedsAttention` 仅驱动顶栏连接环。

<a id="ux-006"></a>
### UX-006 · 同一待审批状态三重呈现 · ✅ 已修

- **问题**：同一实例的"待审批"同时出现在 ① 置顶「需要你 (N)」卡 ② 「状态 · live 会话实例」列表 ③ 工作区行角标——聚合视图与分布视图全部常驻，一屏内同一事实出现 2–3 次，冗余没有换来更快定位。
- **证据**：实测截图（`test:needsyou` 场景侧边栏，Background Approval Demo 出现两次、blog-static ⚠️ 出现两次）。
- **建议**：以「需要你」卡为唯一聚合入口（它最有效、带等待时长），实例列表与其合并或仅在展开时显示。
- **涉及**：`public/js/app.js` sessionPanel。
- **落地**：② 整段移除；待审批只留 ① 需要你卡 + ③ 树行角标。

### UX-007 · 会话列表默认折叠

- **问题**：工作区行 ▸ 默认收起，进一个具体会话 = 开抽屉 → 展开工作区 → 点会话，3 步踩满"3 步找到核心功能"上限。

- **涉及**：`public/js/app.js` 工作区树渲染。

<a id="ux-008"></a>
### UX-008 · 新会话空白态零引导

- **问题**：点 ＋ 新建（最高频入口）后主区完全空白——没有问候、没有能力示例、没有当前工作区/模型确认。问候语 + 最近会话的 dashboard 只存在于"无实例空首页"路径。
- **证据**：实测截图（连接后未发消息的实例视图）。
- **建议**：空会话态复用 dashboard 的极简变体：问候 + 当前工作区/模型一行确认 + 2–3 个与本仓库相关的示例 prompt（点击即填入输入框）。
- **涉及**：`public/js/app.js`（showDashboard / leaveStartScreen 一带）。

<a id="ui-002"></a>
### UI-002 · 发送按钮激活态非品牌色、禁用态过弱 · ✅ 已修

- **问题**：全站最高频 CTA 的激活态是黑色（ink）圆钮，而「允许」「进入」等主 CTA 全是 accent 橙——主动作色不统一（Claude 官方 app 发送钮也是 accent）；disabled 态是 40% 不透明度灰箭头，新手难以发现发送位置。
- **证据**：实测截图（多行输入激活态 vs 空输入禁用态）。
- **建议**：激活态改 accent 底白箭头；disabled 提高到约 60% 不透明度并保留描边感。
- **涉及**：`public/js/app.js` 发送钮状态切换类名、`public/index.html` #btnSend。

<a id="ui-003"></a>
### UI-003 · 代码块复制按钮遮挡内容 · ✅ 已修

- **问题**：`.code-copy-btn` absolute 定位在代码块右上，单行长命令时直接压住尾部代码文字；按钮触点约 22px，手机上难点中。

- **涉及**：`public/css/app.css`。

<a id="ui-004"></a>
### UI-004 · 一批高频小按钮触控目标不达标

- **问题**：44px 热区规范（`.chat-input-pill button::after` 负边距扩充）只覆盖输入区，未覆盖：消息动作栏 复制/朗读/编辑（约 26px 高）、用户气泡复制（约 24px）、code copy（约 22px）、侧栏与文件浏览的 ✕（28px）。
- **证据**：`public/css/app.css`（.msg-action-btn padding 0.25/0.5rem + 11px 字号）、`public/index.html`（w-7 h-7 关闭钮）。
- **建议**：把热区扩充抽成通用类（如 `.hit-44 { position:relative } .hit-44::after { inset:-8px … }`）应用到全部 <44px 可点元素。
- **涉及**：`public/css/app.css`、`public/index.html`、`public/js/app.js` 动态节点。

<a id="ui-005"></a>
### UI-005 · 微型字号泛滥且配踩线色

- **问题**：9.5px（设置磁贴副文案）、10px（RTT/图例/dashboard 标签/控制台 footer）、10.5px（状态胶囊行）、11px（statusline/横幅/说明）遍布全站；中文 10px 级笔画粘连。且这批小字主用 `ink-faint`——对比度 canvas 4.44 / sunk 4.21，双双踩线（AA 4.5），"最小的字配最淡的色"。
- **证据**：`grep 'text-\[9\|text-\[10\|text-\[11' public/index.html public/js/app.js`；对比度计算表（附录）。
- **建议**：非装饰性文字下限 12px；`ink-faint` 退出 <12px 场景，一律升 `ink-soft`（7.58/6.67 余量充足）。空间不够收纳信息而非缩字。
- **涉及**：`public/index.html`、`public/js/app.js`、`public/css/app.css`。

<a id="ux-009"></a>
### UX-009 · 镜像态三枚「CLI 未知」胶囊

- **问题**：会话被终端驾驶（镜像只读）时，三枚状态胶囊变成「CLI 模型未知 / CLI 模式未知 / CLI 档位未知」并排——三个胶囊齐喊"不知道"，信息量为零还占一整行，易被误读为故障。
- **证据**：实测截图（`test:mirror` 场景）。
- **建议**：镜像态下三胶囊合并为一枚「⏱ 终端驾驶中」（点击展开镜像说明），恢复可写后再还原三胶囊。
- **涉及**：`public/js/app.js` 胶囊刷新逻辑（applyMirror 一带）。

<a id="ux-010"></a>
### UX-010 · 输入区上方状态层无仲裁

- **证据**：`grep aria- public/index.html public/js/app.js` 计数；真机截图评价 §7 独立指出同一问题（互相印证）。
- **建议**：分三步——① sheet 家族补 `role="dialog" aria-modal="true"` + 打开时移焦、关闭时还焦（一处基建改动覆盖全家族）；② 消息区加 `aria-live="polite"` 容器（流式 delta 合并播报）；③ 状态 emoji 旁加 `sr-only` 文本或 `aria-label`。对目标画像（自托管 power user）非阻断项，故列 P2，但 ① 成本低值得先做。
- **涉及**：`public/index.html`、`public/js/app.js` sheet 开关逻辑。
