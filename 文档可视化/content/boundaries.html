模块职责分工

每个根级 JS 模块有明确的职责边界。管什么与不管什么同样重要：

模块管什么不管什么
server.jsHTTP/Socket.IO 路由，鉴权闸门三层编排，会话编排，推送通知调度，工作区热加载不处理 SDK 消息映射，不管理 CLI 子进程，不存储消息内容
agent.jsSDK query() 调用，agent:event 映射，权限闸门，后台任务注册表，环形缓冲区不处理 HTTP 请求，不管理会话元数据持久化，不负责前端渲染
sessions.js会话元数据映射持久化到 data/sessions.json，currentByCwd Map，200ms 防抖异步原子写不存储消息内容，不管理 CLI 进程生命周期，不创建或销毁 AgentSession
history.jsCLI JSONL 文件读取，流式解析，基于 mtime 的 LRU 缓存不写 JSONL，不修改 CLI transcript，不提供实时流（实时流由 agent:event 负责）
workdirs.js多工作区白名单解析与校验，workdirs.json 热加载不管理会话，不控制 CLI 工作目录切换
uploads.js附件接收与落盘，路径注入到消息上下文，防穿越（O_NOFOLLOW + symlink 检查）不扫描文件内容，不限制允许上传的文件类型
devices.jsTOFU 设备信赖持久化，trusted-devices.json + pending-devices.json 管理不参与鉴权逻辑决策，不管理 AUTH_TOKEN
cf-access.jsCloudflare Access JWT 校验，JWKS 本地验签，8s 超时不颁发 token，不管理设备信赖，不参与非公网部署的鉴权
file-security.jssymlink 穿越防御，owner-only 权限校验，原子写不参与业务逻辑，不处理文件内容
sanitizer.js日志脱敏（15 种敏感模式匹配替换）不写日志，不改变数据实际内容（仅脱敏日志输出副本）
notifications.js事件到通知文案的纯映射函数，渠道无关文案 + ntfy 元数据不发送推送，不管理 Web Push 订阅
statusline.jsWeb 自有状态栏：SDK usage 读数，git 分支/变更状态，CLI 版本不读 ~/.claude/settings.json，不模拟终端状态行
app.js（前端）UI 渲染，Socket.IO 客户端，agent:event 三层路由分发器，会话 DOM 缓存，离线队列不调用 SDK，不访问文件系统，不存储持久状态
logic.js（前端）纯函数：shouldDropAgentEvent、aggregateStates、resolveEffortSelection不操作 DOM，不发 socket 事件，不做异步操作

非目标清单

多用户

每实例单用户是安全边界。多用户引入会话隔离、权限分级、资源配额等复杂度，与中间层透明原则冲突。多租户中转 SaaS 在架构层面被排除。

速率限制

单用户场景下速率限制的价值有限。CLI 和 Anthropic API 本身有调用频率上限。Web 层不做额外限制，不引入限流中间件。

自建消息数据库

CLI JSONL 是唯一消息事实源。额外数据库引入双写一致性问题，增加运维负担。

中间层智能

系统不做消息摘要、不修改用户输入、不优化 Claude 输出。中间层的角色是管道，不是处理器。

终端模拟器

不模拟终端 ANSI 转义序列、不渲染 CLI 交互界面。Web UI 使用自有状态栏，不读 ~/.claude/settings.json 的终端状态行配置。

原生 App

不做 iOS/Android 原生 App。PWA 覆盖移动端核心需求（添加到主屏幕、离线推送、全屏沉浸）。

Lint 构建流程

纯 JS 项目无 TypeScript 编译、无构建步骤、无打包器。Node ESM 原生支持。前端使用自托管 vendor 依赖。零构建工具链降低了开发和部署门槛。

中间层透明原则

系统不删改、不摘要、不"优化"Claude 的输出。这是一条设计红线。系统的价值在于把 Claude CLI 的能力搬到手机，而不是在中间加一层"智能"。管道不需要脑子。用户在前端看到的内容和 CLI 输出的内容逐字节一致。

此原则的三点推论：

  • 不缓存或聚合模型回复内容。历史回显直接从 CLI JSONL 读取。
  • 不预检或后处理工具调用结果。权限闸门只问"是否允许"，不问"结果是否合理"。
  • 不添加系统提示或注入上下文。用户发的消息逐字到达 CLI，不加前缀、后缀或格式包装。

前端与后端的职责边界

前端（浏览器）

  • 渲染 UI，处理用户触摸/键盘交互
  • 通过 Socket.IO 发送消息和接收 agent:event
  • 管理本地易失状态（输入缓冲区、历史 DOM 缓存、设备 ID、离线队列）
  • Service Worker 仅用于 Web Push 通知，不做离线页面缓存
  • 纯函数（logic.js）保持无副作用，可测试

后端（Node.js 服务）

  • 鉴权闸门（三层：token + 设备信赖 + CF Access JWT）
  • 会话编排和 AgentSession 生命周期管理
  • CLI 子进程创建、通信、销毁
  • 文件系统和 CLI JSONL 的独占读写访问
  • 推送通知调度（Web Push + ntfy）
  • 安全收敛：文件穿越检查、路径注入、日志脱敏

唯一事实源

CLI 会话是唯一消息事实源。服务端只存元数据映射（data/sessions.json），永不存消息内容。历史消息始终从 CLI 的 JSONL 文件读取：~/.claude/projects//.jsonl。sessions.js 的 currentByCwd Map 建立从工作目录到 sessionId 的快速索引。
