部署架构

生产实例由四个组件构成：应用服务（Express 5 + Socket.IO 4，监听 localhost:3000）、常驻进程（macOS LaunchAgent，RunAtLoad + KeepAlive）、隧道（Cloudflare Tunnel 命名隧道，域名流量转发至本机 3000）、鉴权网关（Cloudflare Access JWT 校验，fail-closed）。

公网请求路径：用户 → Cloudflare 边缘 → Access JWT 校验 → Tunnel → localhost:3000 → 应用路由。

隧道搭建

  1. 认证：cloudflared tunnel login——浏览器跳转 Cloudflare 授权
  2. 创建命名隧道：cloudflared tunnel create my-tunnel——生成凭证 JSON
  3. 绑定 DNS：cloudflared tunnel route dns my-tunnel chat.example.com
  4. 配置 ingress：编辑 ~/.cloudflared/config.yml，写入 tunnel/credentials-file/ingress 三段

Access 配置

  1. Cloudflare Zero Trust → Access → Applications → Add → Self-hosted
  2. 填入域名，配置策略（Email OTP 或 Google/Microsoft IdP），指定授权邮箱
  3. 取应用的 AUD tag，连同 team 名填入 .env：CF_ACCESS_HOSTNAME、CF_ACCESS_TEAM、CF_ACCESS_AUD

常驻服务

deploy/ 目录提供两个 LaunchAgent 模板：server.plist.template（应用服务）和 tunnel.plist.template（cloudflared 隧道）。替换占位符后安装：

# 替换占位符
sed -e "s|__LABEL__|com.ccm.server|g" \
    -e "s|__REPO__|$HOME/code/claude-chat-mobile|g" \
    -e "s|__NODE__|$(which node)|g" \
    -e "s|__LOG__|$HOME/Library/Logs/ccm-server.log|g" \
    deploy/server.plist.template > ~/Library/LaunchAgents/com.ccm.server.plist

# 启动
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.ccm.server.plist
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.ccm.tunnel.plist

安全加固清单

  • 收紧 permissions.allow：移除 Bash(*)、rm、sudo、curl 等危险放行
  • 固定 WORK_DIR 到具体项目目录，不使用 $HOME
  • VAPID/Anthropic 凭据只从启动 shell 注入，不写 .env
  • 公网必上 CF Access，禁止裸暴露 3000 端口
  • 配置文件权限 0600（.env、data/cf-access-certs.json 等）
