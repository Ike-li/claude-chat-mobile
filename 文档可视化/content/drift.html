架构层面的注意事项

以下不是"文档错了"，而是在阅读文档时需要知道的背景——文档写于特定时间点，代码在持续演进。

分支模型design.md §7 描述的是 dev → master ff 合并模型。实际代码中 dev 和 master 可能处于不同进度。以 git log 的实际提交历史为准。
SDK 版本文档引用的 SDK 版本号（0.3.201）以 package.json 为准。升级后 API 变化可能导致文档中的代码示例过时。
CLI 行为SDK 对 CLI 的封装层可能因 CLI 版本升级而行为变化。agent.js 中的 map() 映射逻辑是最权威的"SDK 消息 → agent:event"转换定义。
前端组件public/index.html 中的 DOM 结构和 CSS 类名是前端 UI 的权威定义。文档中的 UI 描述以实际 HTML 为准。

核对方法

判断文档是否过时的方法是直接看代码：

  1. 如果文档引用了某个函数/事件/常量，在源码中搜索它——存在且签名一致 → 文档有效。
  2. 如果文档描述了某个行为，找到对应的测试（test/ 或 scripts/smoke-*.js）——测试通过 → 行为有效。
  3. 如果以上两者都找不到，文档可能已过时。以代码为准，并考虑更新本页。
