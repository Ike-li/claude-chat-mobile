信封结构

每个 agent:event 都是固定结构的 JSON 对象，由 agent.js 的 emit(type, payload) 或 emitTransient(type, payload) 创建：

{
  seq:        Number,      // 单调递增序号（环形缓冲 500 条）
  epoch:      Number,      // Date.now() + 计数器（重启保证递增）
  sessionId:  String|null, // 所属会话 ID
  instanceId: String,      // 归属实例 ID（用于多实例分流）
  cwd:        String,      // 工作区绝对路径
  ts:         Number,      // wall clock 时间戳
  type:       String,      // 事件类型（24 种之一）
  payload:    Object       // 事件体，类型取决于 type
}

seq/epoch 机制

seq单调递增整数。每 emit() 一次增加 1（emitTransient 不增 seq）。环形缓冲 500 条上限。seq 溢出后最早事件被覆盖，客户端落后超过 500 条时检测到 gap 并触发全量历史重载。
epochDate.now() 与计数器组合生成。每个 AgentSession 创建时生成新 epoch，重启后一定大于旧实例。前端检测 epoch 变化 → 重置 seq 计数器、清空流缓存、清空权限/问题队列。

emit vs emitTransient

emit（持久事件）

  • 进入环形缓冲（500 条）
  • seq 递增 + 1
  • 客户端重连时通过 sync:since 重放
  • 适用于：所有状态边界事件

emitTransient（瞬态事件）

  • 不进环形缓冲
  • seq 不递增
  • 断开即丢失，不重放
  • 适用于：task_progress 心跳

sync:since 重连协议

客户端重连后发送 sync:since 事件（含 sessionId、lastSeq、instanceId）。服务端响应：{replayed（增量事件数）, gap（缓冲超窗）, found（实例仍在运行）, pending（服务端是否还在等待 SDK result）, epoch}。

前端 syncAckAction 处理：found=false → reload（清屏重载历史）；gap → reload（缓冲超窗，全量重载）；err（仅 probe 路径）→ reconnect（半开死连接，强制重连）；正常 → none（增量经 epoch/seq 去重渲染）。
