定位

单页 PWA，事件驱动 UI，服务端是唯一状态源。几乎所有 UI 状态流都来自服务端事件（instances、init、result、text_delta 等）。客户端是渲染壳，不是状态机。

整体架构

1

index.html

应用壳 + CSS 令牌（--canvas:#F4F3EE, --accent:#C96442）+ PWA 清单。深色模式 @media 查询。Source Serif 4 自托管字体（400/600/italic）。CSP script-src 'self'（无 unsafe-inline）。

2

app.js（~3520 行）

主逻辑 IIFE。Socket.IO 客户端，agent:event 分发器，流式渲染引擎，会话 DOM 缓存（40 条），离线队列。

3

logic.js

纯逻辑函数，零副作用。shouldDropAgentEvent、aggregateStates、resolveEffortSelection 等。供 node --test 单测。

4

sw.js + vendor/

SW 仅用于 Web Push 通知，不做缓存。vendor/ 自托管：marked v12、DOMPurify v3.4、highlight.js v11.11、Tailwind CSS。

agent:event 三层层路由

  1. OOB 旁路：task_notification、task_progress、history_append、mirror_state——无条件处理，不进后续路由。task_notification 触发 OS 通知和角标。
  2. shouldDropAgentEvent：instanceId 不匹配 viewingInstanceId 则丢弃。合成事件（无 instanceId）和 instances 事件始终放行。instancesReady 为 false 时所有事件放行。
  3. epoch/seq 去重：新 epoch 重置 curEpoch + lastSeq + 清空权限/问题队列。seq ≤ lastSeq 的事件丢弃。

流式渲染机制

  • streams Map：messageId → {el, raw, textNode, done}。惰性创建 TextNode 用于 text_delta 追加。
  • getStream(id)：惰性创建消息包装器。text_delta 追加到 TextNode。
  • finalizeStreams()：result 到达时，取所有流的累积 raw 文本，全量 marked + DOMPurify + highlight.js 渲染后替换 innerHTML。保证跨重连、跨丢包的完整 Markdown 渲染。
  • thinking：别建 collapsible details.thinking 元素，斜体衬线字体。

权限审批流程

队列型：permQueue 持有待处理权限。activePerm 守卫：一次只展示一个。showNextPerm()：从队列取，填入工具名、cwd、完整 input JSON（超 4000 字符可展开）。answerPerm(decision)：emit user:approve，关闭弹窗，展示下一个。跨设备一致性：另一设备解决同一请求时，request_resolved 关闭陈旧弹窗。

会话 DOM 缓存

sessionDomCache：Map(sessionId → {nodes, lastSeq, epoch, instanceId})，上限 40 条。bindView 切换时先查缓存——命中则 0ms 恢复 DOM 并从 cachedSeq 做 sync:since 增量同步。clearView：清 DOM、streams、队列、关闭弹窗、停止 TTL 计时器。

离线恢复

  • 离线队列：socket 断开时消息存 offlineQueue，乐观 UI（opacity-70 + pending 指示器）。重连时 processOfflineQueue 重放。
  • foregroundReconnectAction：半开连接检测——socket.connected 但实际可能是僵尸连接（JS 在后台被冻结），发 sync:since 探测（5s 超时）。超时 → 全量重连；found=false → 重载历史。
  • sync:since：断线重连后按 lastSeq 增量补发。gap → 历史重载。

其他 UI 特性

  • TTS 语音朗读（browser SpeechSynthesis，每条消息独立 speak/stop）
  • 代码块复制按钮（hover 显示，navigator.clipboard）
  • 斜杠命令提示（/ 触发下拉，来自 init 事件的 slash_commands）
  • 触觉反馈（navigator.vibrate，tap/success/error/warning 四种模式）
  • 深链解析（notificationclick → postMessage 或 URL hash → applyDeepLink → setViewing 或 session:switch）
  • iOS 适配：keyboardInsetPadding（visualViewport 计算，inputFocused 守卫）、safe-area-inset-*
  • 侧栏手势：边缘左滑打开、左滑关闭、行右滑露出关闭按钮
