项目本质

本项目本质上是一条把本机 shell 暴露到公网的代码执行通道。每个自托管部署的用户即部署者、即管理员；通过鉴权的请求拥有与"本人坐在终端前"完全相同的权力。这不是漏洞——漏洞是"未通过鉴权的人获得这个权力"。

项目一旦对外暴露（如通过 Cloudflare Tunnel），即成为暴露在公网上的任意代码执行通道。安全模型由此而来：一切防护围绕"鉴权边界"展开，而不是限制已鉴权用户能做什么——终端本身也不会有这层限制。

安全不变量

以下 11 条不变量是安全模型的稳定性边界。违反即回退。

#不变量说明
1不设 token 则不出本机未设置 AUTH_TOKEN 时服务只监听 127.0.0.1 并在启动时醒目警告。要对外服务必须设 token。不存在留空等于公网裸奔的路径。
2token 卫生token 经 /#token=值首次注入后立即从地址栏清除并存 localStorage；等值比较用 crypto.timingSafeEqual，防计时侧信道。
3审批信息完整性审批弹窗必须展示完整命令 + cwd + 关键参数，不得只显示工具名。移动端审批的上下文密度低于终端，信息完整性是硬约束。
4透明性中间层不加工 claude 的语义输出（不删改、不摘要、不优化）。上下文管理是 claude 自己的事。
5物理性任务生命周期大于等于连接生命周期（socket 断开不中断任务）。等待用户输入时不超时。agent 静默超过 IDLE_TIMEOUT_MS（默认 600000ms）判为挂死，中断并报告。
6bypassPermissions 护栏默认关闭、无自动进入路径、须 UI 显式切换加一次危险确认。开启即接受爆炸半径等于整机。代码里不得加任何自动进入 bypass 的路径。
7公网 fail-closed启用 Cloudflare Access 时，经 CF 的公网请求强制验签 Access JWT、不接受 token 回退。按 Host 判定鉴权，堵死不发头改走 token 路的做法。
8不开无鉴权数据端点历史回显走鉴权的 session:history socket 事件；/health 在设 token 时也需鉴权——避免对外泄漏运行状态。
9TOFU 设备信赖非本地且非 CF Access 验证通过的连接，必须通过机主在主机端的一次性显式授权。未授权连接的所有上行 Socket 事件均被拦截丢弃，从 server.js 统一事件过滤点 fail-closed 执行。
10预览只读且不越界tool:preview 经 attributePath 唯一闸门（路径归属加 symlink 加 realpath 二核）只读白名单工作目录内文件；即便 claude 曾按 permissions.allow 读过白名单外文件，预览一概拒绝。
11诊断输出脱敏doctor:run 安全体检只回显布尔值、计数、危险规则串，绝不外泄明文 token、密钥、绝对路径或 AUD——体检是防御工具，其报告本身不得成为新的泄露面。

鉴权分层

鉴权由三层构成，层层递进、不替代：

第 1 层：httpAuth按 Host 判定鉴权方式：公网请求（Host 匹配 CF_ACCESS_HOSTNAME）强制验签 Cloudflare Access JWT；局域网或未启用 CF Access 的请求检查 AUTH_TOKEN。未设 token 时服务只监听 127.0.0.1。
第 2 层：TOFU 设备信赖非 localhost 且非 CF Access 验证通过的连接，必须经机主在主机端一次性显式授权。未授权设备的所有上行 Socket 事件被统一拦截丢弃。
第 3 层：Cloudflare Access JWT纵深防御层。通过隧道的公网请求强制带合法 Access JWT，使用 jose 本地 JWKS 验签，8s 超时（适配 Clash 直连场景）、30s 防重放冷却。fail-closed：验签失败即拒绝，不接受回退至 token 鉴权。

bypassPermissions 的特殊风险

bypassPermissions 档下审批弹窗完全失效——开启即接受爆炸半径等于整机。代码里不得加任何自动进入 bypass 的路径。必须 UI 显式切换加一次危险确认才能开启。该档适用于开发环境或完全可信任的网络，不应用于公网暴露的场景。

部署加固建议

  • 公网常驻部署应配置 Cloudflare Access 双因素认证，不要长期裸用随机隧道。
  • 收紧 ~/.claude/settings.json 的 permissions.allow：移除 Write、Bash(npm run *) 等可组合成自放行链的项。
  • 把 WORK_DIR 固定到具体项目目录，不要使用 $HOME。
  • VAPID 和 Anthropic 凭据只从启动 shell 注入，不写入 .env 文件。
