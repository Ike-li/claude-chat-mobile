两个模块的分工

sessions.js（146 行）

管元数据：谁、什么时候、哪个工作区。持久化到 data/sessions.json，永不存消息内容。currentByCwd Map 建立从工作目录到 sessionId 的快速索引。

history.js（291 行）

管历史内容：说了什么。从 CLI JSONL 文件（~/.claude/projects/）流式读取，基于 mtime 的 LRU 缓存。是前端展示历史消息的唯一数据来源。

消息内容不存服务端：CLI JSONL 是唯一事实源。这条约束来自终端等价性：CLI 没有自己的消息数据库，服务端也不应该有。元数据仅用于会话列表显示和 resume 时恢复档位。

sessions.js 数据模型

currentByCwdMap：每工作目录一个当前会话指针。缺省或记录过期时返回 null。
sessions会话元数据数组。每条包含 id、title、cwd、model、effort、permissionMode、createdAt、lastUsedAt。永不存消息内容。

upsertSession 逻辑

  • 已存在（按 id 匹配）：更新 lastUsedAt；同步 model、effort、permissionMode（diff 更新，不覆盖显式 null）。
  • 新条目：unshift 到 state.sessions 头部。记录创建与最后使用时间戳。title 从入参获取或占位为"新会话"。
  • 两种路径均设 currentByCwd[cwd] = id。

持久化：200ms 防抖异步写入，严格原子写（writeFile(tmp, 0o600) → rename(tmp, FILE)）。关闭时调用 flushSaveSync() 同步写入。

history.js 读取流程

  1. getProjectDir(cwd)：根据 cwd 推断 CLI 项目目录名。路径中所有非字母数字字符替换为 -。cwd 须先经 realpathSync 规范化，才与 CLI 命名一致（如 /tmp → /private/tmp）。
  2. listSessionsPage(cwd, {limit})：readdir → stat（并发 allSettled）→ 按 mtime 降序 → 前 limit 个并发 readHeadMeta → 返回 sessions + hasMore。
  3. readHeadMeta(file, size)：读文件头部 64KB，逐行 JSON parse。提取 ai-title、model、entrypoint。头部漏读时补读尾部 512KB（含 4KB 跨边重叠）。
  4. getSessionHistory(sessionId, cwd, limit)：createReadStream + readline 逐行流式读取——不一次性 buffer 整个文件。逐行解析 → 过滤 → 提取文本 → 封顶到 2000 条。

过滤规则详解

过滤规则过滤对象理由
isMetaCLI 注入的元信息行非用户或 assistant 对话内容，回显会制造大量无用堆砌
isSidechain子 agent 的记录与运行期 agent.js 的 parent_tool_use_id 守卫等效。终端行为等价
parent_tool_use_id带此字段的行（防御性）isSidechain 的防御补充，确保子 agent 消息绝不进入主消息流
纯工具块content 全为 tool_use/tool_result 的条目空文本工具调用渲染成空气泡占用窗口
task-notification 注入CLI 向消息流注入的系统信号内部控制信号，非用户对话内容

readHeadMeta 标题提取优先级

  1. ai-title（CLI 自动生成）：扫描头部 64KB，取最后一个（后写更准）。未找到且文件大于 64KB 时补读尾部 512KB。
  2. 首条用户文本：第一条非命令包裹的真实 user 消息。
  3. 首条命令名：仅当 ai-title 和首条用户文本均缺失时的回退。

缓存策略

history 缓存按文件路径缓存 getSessionHistory 结果。基于 mtimeMs 失效。LRU 淘汰上限 10 条。
list 缓存按 {dir:limit} 键缓存 listSessionsPage 结果。4s TTL。新会话创建后立即失效缓存。

catchUpStep 状态机

纯函数实现（便于单测）。用于 web 端续接终端 CLI 里正在跑的会话：

localBusy 抑制本地实例正在跑 turn → 只记 wasBusy=true，不推，不动 baseline。
busy → idle 重置刚结束自己的 turn → 增量归因于自己刚写盘，只重置 baseline 到当前长度。
idle 增长推送持续 idle 期间 JSONL 增长 → 判为外部（终端）写入 → 推超出 baseline 的尾部消息。

sessionFileExists 安全校验

  • 字符集校验：正则 ^[0-9a-zA-Z_-]+$ 确保 id 只包含合法 session id 字符。拒绝含 /、\、. 的任何输入。
  • 文件存在性检查：校验通过后 stat 验证 JSONL 确实存在于预期的 project 目录内。
  • 任何校验失败 return false：不允许 fallback、静默降级或重试。
