唯一判据

项目全部需求的唯一判据是终端等价性：对任何一次交互，问"此刻我若坐在终端前，会得到什么？"移动端必须给出语义等价物。形态可以不同——聊天气泡代替 TUI，弹窗代替 y/n——但等价物必须存在。

判据既是下界也是上界。少于它等于残缺（claude 在终端能改文件跑命令，移动端只能聊天则不合格）；多于它等于膨胀（多用户、速率限制、自建消息数据库——终端没有这些，本项目也不许有）。

时间维度

等价性不仅约束"注视时刻看到什么"，也约束"agent 需要我"到"我知道"的延迟。坐在终端前时这个延迟近似为 0；移动端注意力是间歇的，没有主动触达就可能一直等下去。E15（推送触达）和 E8（交互式问题）因此成为 P0，而非可选的锦上添花。

对象维度

等价的对象是 agent 能力——对话、工具、技能、审批、上下文管理、resume、模型切换——SDK 全部暴露，init 事件的 slash_commands 是其机器可读边界。不包括终端客户端自己的 UI（/plugin、/config 等面板）。后者用 app 自身的 UI 或自然语言替代，不复刻终端面板。

等价性如何驱动架构决策

E10 断线重连 → 环形缓冲 + seq/epoch + sync:since

终端不会因为你走开就断开连接。移动端解决了"断线是主路径而非异常分支"的问题——环形缓冲区 500 条上限 + epoch（Date.now() + 计数器）+ seq 单调递增 + sync:since 增量补发 + gap 检测。连接断开后任务继续运行，重连后按 lastSeq 补发错过的全部事件。前端分流不丢后台实例的输出。

E15 推送触达 → notificationForEvent 纯函数 + 双渠道通知

坐在终端前时 agent 需要你即被看到。notificationForEvent 纯函数映射事件类型到推送文案，双渠道并发——Web Push（浏览器标准推送）和 ntfy（HTTP POST，绕过 iOS 加主屏与局域网 http 的双向约束）。permission_request、question 无条件推送；result 仅在无客户端连接时推送。

E1 会话连续 → CLI JSONL 是唯一事实源

sessionId 不在服务端生成，不在数据库中注册——它来自 CLI resume 返回的真实 session_id。服务端只存元数据映射（cwd、标题、模型、权限档），绝不存消息内容。

E14 历史回显 → 流式读 JSONL + 2000 条防爆

不使用自建消息数据库。切换会话时流式读取 CLI 的 JSONL 文件（createReadStream + readline），逐行解析过滤后返回。2000 条防爆上限，与 /resume 同源。
