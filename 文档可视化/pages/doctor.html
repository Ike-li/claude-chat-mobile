配置自检与安全体检

doctor.js 启动前 10 项检查 + UI 内安全体检的完整清单。

阅读约 8 分钟 第 23 / 27 篇 源自 scripts/doctor.js · doctor-runtime.js

两类检查

启动前自检（doctor.js）

node scripts/doctor.js 执行 10 项检查。全部通过后方可正常启动。阻塞启动流程。支持 --env=path 指定 .env 文件、--fix 自动修复权限。

UI 安全体检（doctor-runtime.js）

在应用内设置页面以只读方式运行。6+1 项检查指标。不修改任何文件。全程脱敏——只出布尔/计数/危险规则串。

doctor.js 10 项检查

#检查项检查方式不通过影响
D1AUTH_TOKEN 非空且合理读取 .env；校验长度和字符范围启动失败——无有效令牌
D2CLAUDE_BIN 可执行fs.access 检查可执行权限claude CLI 无法调用
D3WORK_DIR(S) 可写fs.access W_OK会话无法创建文件
D4PORT 未被占用net.createServer().listen() 探测服务启动失败——端口冲突
D5WEB_STATUSLINE 配置检查环境变量状态栏不可用（不影响核心通信）
D6网关环境一致性检查 .env 中 ANTHROPIC_* 变量这些变量应在启动 shell 注入
D7配置文件权限fs.stat 检查是否为 0600敏感文件可能被其他进程读取
D8文档一致性委托 doc-consistency.js文档与代码不同步
D9前端 JS 语法node --check public/js/*.js前端脚本语法错误
D10测试覆盖率阈值读取覆盖率报告；默认 65%CI 门禁不通过

doctor-runtime.js 6+1 检查

readMergedPermissions合并三层 permissions.allow：global（~/.claude/settings.json）→ project（.claude/settings.json）→ local（.claude/settings.local.json）。坏 JSON skip 不清空。

检查清单：AUTH_TOKEN（是否设置及长度）→ CLAUDE_BIN（版本是否可采集）→ WORK_DIRS（数量，不泄露路径）→ CONFIG_PERMS（配置文件 0600）→ CF_ACCESS（是否启用，AUD 仅布尔）→ PUSH_VAPID（是否配置）→ DEVICES（受信/待审批设备数）→ WHITELIST（summarizeDangerous 识别危险放行 + scope 标注）。

computeReadiness 综合判定：全部 ok → ready；有 warn 无 blocked → caution；有 blocked → blocked。

白名单审计

summarizeDangerous 识别危险放行：Bash(*)（任意命令执行）、rm（文件删除）、sudo（提权执行）、curl/wget（网络请求/文件下载）、chmod/chown（权限/属主变更）。每条标注 scope（global/project/local）。

全程脱敏原则：只输出布尔值、计数和危险规则字符串。绝不回显明文 token、密钥、绝对路径或 AUD tag。体检是防御工具，其报告本身不得成为新的泄露面。
