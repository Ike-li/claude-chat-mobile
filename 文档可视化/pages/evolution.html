架构演进主线

从本地 CLI 到公网移动端的逐层演进，关键决策点回顾。

阅读约 12 分钟 第 6 / 27 篇 源自 git history · docs/design.md

演进五阶段

系统从本地 CLI 工具逐步演进为公网 Web 服务。每个阶段解决一组问题，也引入新问题。

1

本地 CLI

直接在终端运行 claude。零隔离、零网络依赖。问题：离开电脑就无法使用，无法在移动中继续对话。

2

本地 server

Express 服务在 localhost:3000 启动，手机和电脑在同一局域网下访问。解决了移动访问问题。引入问题：HTTP 端点暴露在局域网中、无任何鉴权、无会话持久化、仅支持单工作区。

3

局域网 + 基础设施

添加 AUTH_TOKEN 鉴权、sessions.js 会话持久化、statusline.js 状态栏。支持同一个 Mac 上多个项目用不同的工作区。引入问题：只能在家用或办公网络使用。离开局域网范围无法访问。

4

临时公网隧道

cloudflared 临时隧道将 localhost:3000 暴露到公网。在外网也能访问。引入问题：隧道 URL 每次启动变化、无身份鉴别的公网暴露（token 再强，URL 被扫描就是暴露面）。

5

固定域名 + Access 2FA

Cloudflare Tunnel 固定域名 + Cloudflare Access 双因素认证。稳定的自定义域名。OTP 二次验证保护公网入口。引入问题：运维复杂度上升、Cloudflare 单点依赖、域名和证书管理成本。

架构决策转折点

转折之前之后触发因素
会话索引全局 currentSessionIdcurrentByCwd Map（每个工作区独立会话）多工作区需求。多个项目共享一个服务实例时，同一 cwd 复用会话，不同 cwd 各自独立。
工作区管理单 WORK_DIR 环境变量WORK_DIRS 白名单 + workdirs.json 热加载项目增多后需要动态增删工作区，重启服务影响在线对话。热加载通过文件监听实现。
AgentSession 模型单实例多 AgentSession 并发（每 tab 独立）多 tab 需求。每个 tab 调用独立的 claude --resume 子进程，互不干扰。
历史端点HTTP GET /historySocket.IO session:history_append 事件减少 HTTP 公网端口暴露面。与现有 Socket.IO 鉴权路径一致，支持增量推送。
事件编码无统一信封agent:event 统一信封（epoch + seq 去重）多实例并发后需要跨实例事件顺序和去重机制。500 条环形缓冲区上限防止内存泄漏。
只读锁策略mtime 预锁运行时观察式锁定web resume 自己刷 transcript mtime 致纯 Web 打开被误锁不能输入。改为切入不预锁，仅运行期观察到新消息才锁。

安全模型逐层加厚

1

纯 token

AUTH_TOKEN 环境变量。Bearer token 鉴权。最简单形式。问题：token 泄露即为完全暴露。无设备粒度识别。

2

TOFU 设备信赖

首次连接时生成设备指纹，存入 pending-devices.json 等待审批。admin 通过 approve/deny 管理设备白名单。新设备即使持有 token 也无法自动接入。

3

CF Access JWT

部署在 Cloudflare Tunnel 后方时，Cloudflare Access 在请求到达服务端之前校验用户身份。服务端对 Cf-Access-Jwt-Assertion 头做 JWKS 本地验签（8s 超时）。公网部署时提供 OTP 双因素认证层。

三层鉴权严格采用 fail-closed 设计。任何一层校验失败都断开连接，不降级。公网部署下三层同时生效；局域网部署下仅 token + 设备信赖两层。

前端演进

1

基础聊天

简单消息列表 + 输入框。Socket.IO 连接。无状态栏、无上传、无权限 UI。只能收发纯文本消息。

2

状态栏

statusline.js 驱动。展示 SDK usage（输入/输出 token 计数）、git 分支与文件变更数、CLI 版本号。Web 自有实现，不读 ~/.claude/settings.json。

3

文件上传与预览

uploads.js 接收附件并落盘。file-preview.js 提供工具卡片文件预览。路径穿越防御（O_NOFOLLOW + symlink 检查）。

4

TOFU 设备审批 UI

等待确认设备列表。approve/deny 操作直接在界面完成，不需要 ssh 到服务端。

5

完整 PWA

PWA maskable 图标、离线 Web Push 通知、深色主题、自适应主题色。自托管 Source Serif 4 字体。Service Worker 仅用于 Web Push。前端 40 条 DOM 缓存上限。
