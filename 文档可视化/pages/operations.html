运维与故障排查

日常运维命令、常见故障诊断、日志解读。

阅读约 10 分钟 第 22 / 27 篇 源自 docs/deployment.md

日常运维命令速查

操作命令
启动常驻服务launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/<label>.plist
重启常驻服务launchctl kickstart -k gui/$(id -u)/<label>
停止常驻服务launchctl bootout gui/$(id -u)/<label>
查询进程状态launchctl list <label> | grep -E '"PID"|LastExitStatus'
查看日志tail -f ~/Library/Logs/<log>.log
健康检查curl http://localhost:3000/health?token=<TOKEN>

故障排查表

现象处理步骤
公网 502/1033检查 server 和 tunnel 是否在跑：launchctl list
OTP 过了但连不上JWT 校验失败。核对 CF_ACCESS_TEAM/AUD 与 Zero Trust 面板是否一致
手机进不去登录页检查 DNS 解析 + tunnel 日志
改 .env 不生效忘了重启 server 进程：launchctl kickstart
公网 1033 + 全局代理Clash TUN 劫持 cloudflared。方案：edge-bind-address 绑定 en0 IP + 静态路由直连 CF 边缘段
经第三方网关 model_not_found模型名需后缀标记（如 [1m]）
回复只有工具卡片无正文网关不流式。agent.js map() 有全文兜底逻辑
健康检查返回 401配置了 AUTH_TOKEN 后需带 ?token= 或 x-auth-token 请求头

重要提醒

生产实例端口（3000）由常驻服务占用。不要手动执行 npm start——会因端口冲突失败。改 .env 或更新代码后，须重启常驻 server 进程才生效。workdirs.json 例外：server 监听文件变化，改完即生效、免重启（被移除目录上的已开会话继续运行，仅拒新开）。

临时本地调试流程

  1. 停常驻服务：launchctl bootout gui/$(id -u)/com.ccm.server
  2. 前台启动：npm start（或 npm run dev 带 watch）
  3. 调完恢复：停前台进程 → launchctl bootstrap ...

日志位置（macOS）：应用服务 ~/Library/Logs/ccm-server.log；隧道 ~/Library/Logs/ccm-tunnel.log。
