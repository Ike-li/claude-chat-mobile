01 第一部分 · 认识项目

项目总览

定位、核心理念、技术栈一览。

阅读约 10 分钟 第 3 / 27 篇 源自 README.md · CLAUDE.md · docs/capabilities.md

Claude Chat Mobile 是一个自托管的移动端聊天 Web UI，把本机的 Claude Code CLI 接到手机上。核心目标：终端等价性——"坐在电脑前对 claude 打字"和"在手机上打字"，效果一样。

一句话定位

它不是新的 AI，不是 Claude 的重新实现，也不是远程桌面。它是你本机 Claude Code 的一个移动端入口——同一个 agent、同一份 CLAUDE.md、同样的 MCP 服务器、技能、hooks 和已登录会话。

核心理念

项目的设计围绕一个判据展开：终端等价性。对任何一次交互，问"此刻我若坐在终端前，会得到什么？"移动端必须给出语义等价物。

这个判据既是下界也是上界：

少于它 = 残缺

claude 在终端能改文件跑命令，移动端只能聊天——不合格。必须完整支持：流式输出、工具调用、权限审批、会话连续、中断恢复、模型切换。

多于它 = 膨胀

多用户、速率限制、自建消息数据库——终端没有这些，本项目也不许有。中间层只转发，不加工 claude 的语义输出。

用户与信任模型

  • 每实例单用户：每个自托管部署的使用者就是部署者。项目面向所有人开放自托管，但单个实例被设计为一个人用。
  • 机主即 root：通过鉴权的请求拥有与"本人坐在终端前"完全相同的权力。这是产品目的，不是漏洞；漏洞是"未通过鉴权的人获得这个权力"。
  • 不做 SaaS / 多租户：单个实例不做多用户、用户名体系、配额。那是另一类产品。

移动端独有的四个问题

终端没有、但移动端必须解决的四个约束：

公网暴露

终端的信任边界是"物理坐在机器前"；隧道把边界搬到公网。安全优先级最高。

会话碎片化

手机使用模式："看 30 秒 → 锁屏 → 10 分钟后再看"。断线重连是主路径，不是异常分支。

小屏与弱网

渲染与传输效率必须优化。代码块横向滚动、事件增量传输。

注意力非对称

agent 可以一直等，人不一定知道它在等。不超时 + 无触达 = 无界死等。推送通知是必须的。

运行方式

方式适用场景说明
本机直接开发调试npm start，访问 http://localhost:3000
局域网家里 / 办公室连 Wi-Fi 访问 http://<lan-ip>:3000/#token=…
公网（推荐）随时随地Cloudflare Tunnel 固定域名 + Access 双因素
公网（临时）测试cloudflared tunnel --url http://localhost:3000

技术栈一览

后端运行时Node.js ≥20，纯 ESM（"type": "module"），零构建、零打包器
HTTP + WebSocketExpress 5 + Socket.IO 4
Agent SDK@anthropic-ai/claude-agent-sdk 0.3.201（官方 SDK，streaming input 模式）
鉴权AUTH_TOKEN + TOFU 设备信赖 + Cloudflare Access JWT（可选纵深防御）
推送Web Push API（VAPID）+ ntfy（可选，更可靠的移动端推送）
前端纯 HTML/CSS/JS PWA，marked + DOMPurify + highlight.js，Source Serif 4 正文字体
测试node --test（单测，零 token）+ Puppeteer（视觉 E2E）+ 冒烟脚本（真实 token）
许可证AGPL-3.0-only
