阅读顺序指南

三条阅读路线：速览（30 分钟）、完整通读（2–3 小时）、按角色切入。

阅读约 8 分钟 第 2 / 27 篇

这本手册按认知曲线组织：认识 → 演进 → 方法论 → 实现 → 数据 → 运维 → 规范。这不是文件目录的顺序，而是人理解一个新项目最自然的顺序。

路线一：30 分钟速览

适合：想快速了解这个项目是干什么的、怎么搭的、能不能用。

1

项目总览

5 分钟。搞清楚定位、核心理念，一句话：把本机 Claude Code 接到手机上。

2

系统架构总览

5 分钟。三层架构：前端 PWA → Express + Socket.IO 中枢 → Claude Agent SDK。一张图看懂数据怎么流。

3

终端等价性原则

5 分钟。唯一的判据是什么，19 条功能需求（E1–E19）的完整清单。

4

安全模型

5 分钟。11 条安全不变量，威胁矩阵，鉴权分层。项目本质上是一条把本机 shell 暴露到公网的通道——安全是最重要的。

5

快速开始

5 分钟。从零到手机访问的最短路径。

6

漂移清单

5 分钟。文档与代码不一致的地方，接手必读。

路线二：完整通读（2–3 小时）

适合：要接手开发、深入理解每个模块、或基于这个项目做二次开发。

按左侧目录从上到下顺序读。主线是：

  1. 第一部分 · 认识项目：总览 → 架构 → 边界。建立全局心智模型。
  2. 第二部分 · 演进历程：理解今天的架构为什么长这样，每个关键决策的背景。
  3. 第三部分 · 方法论：终端等价性、安全模型、移动端优先——三个不可动摇的设计支柱。
  4. 第四部分 · 核心实现：逐个模块拆解。server → agent → 会话管理 → 前端 → 基础设施 → 安全模块。这是最长的部分，每页 10–14 分钟。
  5. 第五部分 · 数据与集成：事件契约、数据流、外部系统对接。
  6. 第六部分 · 部署与运维：怎么跑起来、怎么跑稳、出问题怎么查。
  7. 第七部分 · 参考与规范：API 速查、开发约定、测试策略、漂移清单。

路线三：按角色切入

角色先读再补
后端开发者系统架构 → server.js → agent.js → 会话管理前端应用、安全模块
前端开发者前端应用 → 事件系统 → 数据流系统架构、server.js
运维 / DevOps快速开始 → 生产部署 → 运维与故障排查 → 配置自检安全模型、外部集成
安全审计安全模型 → 安全模块 → 外部集成 → 漂移清单系统架构、事件系统
