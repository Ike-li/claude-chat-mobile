服务端中枢：server.js

Express + Socket.IO 事件契约层——多实例编排、鉴权闸门、推送与通知。

阅读约 14 分钟 第 11 / 27 篇 源自 server.js

定位

server.js（1766 行）是 Express 5 + Socket.IO 4 构成的事件契约层，扮演多设备 Web UI 与 Claude Code CLI 进程之间的枢纽。不依赖任何构建工具或框架，纯 JS ESM。

启动流程

  1. preflight()：验证 WORK_DIR 存在且为目录（realpathSync 规范化），解析 WORK_DIRS 白名单（支持 JSON 文件或逗号分隔变量），查找本机 claude 命令（E9：必须用本机 claude，不用 SDK 捆绑副本），采集 SDK 与 CLI 版本。
  2. applyWorkdirs()：加载工作目录白名单，条目支持 {path, sessionLimit} 配置每区会话显示条数。WORK_DIR 恒为首位。
  3. httpAuth 中间件注册：注册 Express 全局鉴权中间件（按 Host 判定策略：公网 CF JWT 或局域网 token）。
  4. Socket.IO 连接处理：注册 io.on('connection') 处理入向事件，每连接分配 deviceToken，执行设备信赖校验。
  5. catchUpTick 定时器启动：每 2.5s 轮询 CLI transcript 文件，检测外部写入，通过 history_append 事件推送给前端，并维护 mirrorReadonly 只读锁。

多实例编排

openInstance({cwd, resumeId, mode, effort})创建 AgentSession，分配 inst_N 的 instanceId。档位优先级：显式入参 > pending 预设 > resume 持久化值 > 继承。fresh 会话用 CLI 启动默认。
disposeInstance(instanceId)调用 AgentSession.dispose()，从 agents Map 删除。若 dispose 的是当前查看 tab，viewingInstanceId 回退到任一存活实例。session 留盘可再开。
broadcastInstances()广播完整实例列表至所有客户端，附带各实例的角标状态（忙碌、有审批、后台任务、轮次完成、失败）。

Socket.IO 入向事件处理

事件说明
user:message校验非空、text ≤ 50000、附件大小；无实例则懒开一个；发消息
user:approve / user:answer审批权限请求 / 回答 AskUserQuestion
user:interrupt中断当前轮次，pendingTurns 会计
user:setPermissionMode / user:setEffort切换权限档 / 置换实例切思考强度
user:setWorkdir / user:setViewing切换工作区（白名单校验）/ 切换查看 tab
session:new / session:switch / session:close新建会话 / 按 JSONL 存在性切换 / 释放实例
session:list / session:history按 cwd 返回会话列表 / 流式读 JSONL 回显历史
sync:since断线重连增量补发环形缓冲；found:false 时客户端清屏重载历史
tool:preview工具卡片文件预览，attributePath 唯一闸门防越界读
doctor:run运行时安全体检，全程脱敏
user:approveDevice / user:denyDevice远程批准/拒绝待信任设备

鉴权流程

  1. httpAuth（Host 判定）：传入请求的 Host 匹配 CF_ACCESS_HOSTNAME 时强制验证 Cloudflare Access JWT（fail-closed）；否则检查 x-auth-token 头或 ?token= 查询参数。AUTH_TOKEN 未设时监听 127.0.0.1。
  2. TOFU 设备信赖：连接握手后检查 deviceToken 是否在受信列表。不在受信列表且非 localhost 非 CF Access 通道的，标记为待确认——该连接的所有上行 Socket 事件被统一拦截丢弃。
  3. 事件级拦截：server.js 统一事件过滤点对所有入向事件执行 deviceApproved 校验。非受信设备的操作事件全部无声丢弃。

通知渠道

pushNotify（Web Push）遍历 pushSubscriptions 数组（多设备支持），调用 webpush.sendNotification()。收到 410/404 时单独剔除订阅。permission_request、question、task_notification 无条件推送；result 仅在无客户端连接时推送。
ntfyNotify（HTTP POST）通过 NTFY_URL + NTFY_TOPIC POST 通知到 ntfy 服务。不依赖浏览器推送基础设施（绕过 iOS 加主屏与局域网 http 的双向约束）。支持 click 深链回会话。

关键设计决策

每 tab 一个 AgentSession各 tab 并行存活，互不中断。同仓库可开多会话并发（一个跑长任务、另一个调试），权限档独立。这是终端做不到的能力增强。
前端/后台分流后台 tab 的高频 delta 不广播但仍入环形缓冲——节省带宽的同时不丢失输出，切回时按 sync:since 完整回放。STATE_BOUNDARY 事件始终广播，维持角标和推送的实时性。
不开 HTTP 数据端点所有数据路径统一走 Socket.IO 鉴权通道，避免 HTTP 无鉴权端点成为信息泄漏面。
任务独立于连接存活物理不变量：任务生命周期 ≥ 连接生命周期。Socket 断开不中断 agent 进程。
