测试体系

零 token 契约测试 + 真实 token 冒烟测试 + 可视化 E2E——三层测试策略。

阅读约 10 分钟 第 26 / 27 篇 源自 test/ · scripts/smoke-*.js · scripts/visual-e2e-runner.js

三层测试策略

单元测试

零 token，纯逻辑。node --test 运行 25 个文件，覆盖全部纯逻辑模块。耗时 < 1 秒。无外部依赖。

可视化 E2E

零 token，mock server。Puppeteer 驱动 17 个测试用例（TC-1 至 TC-17），覆盖完整前端交互路径。visual-mock-server.js（2860 行）完整模拟 Socket.IO 协议。

冒烟测试

从零 token 契约验证到真实 token E2E。14 个 smoke-*.js 脚本，覆盖会话隔离、断线重连、权限模式、effort、上传、状态栏、后台任务、并发等关键路径。

单元测试

25 个测试文件覆盖：agent、cf-access、devices、doctor-checks、doctor-runtime、file-preview、file-security、history、logic、models-cache、notifications、sanitizer、sessions、setup、statusline、uploads、workdirs 等全部纯逻辑模块。执行命令：node --test test/*.test.mjs（排除 test/integration/）。

可视化 E2E

  • mock 服务端：scripts/visual-mock-server.js（2860 行），完整 Socket.IO 协议模拟，40+ 注册场景
  • 测试运行器：scripts/visual-e2e-runner.js，17 个测试用例（TC-1 至 TC-17）
  • 驱动：Puppeteer，零外部依赖
  • 执行：npm run test:visual

17 个测试用例覆盖：流式渲染、工具卡片折叠、权限审批、AskUserQuestion 选择、StatusLine 投影、设置面板、跨 tab 权限清除、TOFU 设备信任、空状态、侧栏与会话列表、输入区交互、设置面板全模式、模态面板、乐观 UI、ExitPlanMode 审批、后台任务进度、sync:since 快照对账。

冒烟测试

CI 集成

.github/workflows/test.yml：push 到 master/dev 及所有 PR 触发。Job 1：unit-test（npm test），Job 2：visual-e2e（npm run test:visual）。两 job 均通过后方可合并。需真 Claude 的集成测试用 RUN_CLAUDE_INTEGRATION=1 opt-in。

覆盖率与契约

  • 覆盖率：npm run test:coverage（coverage-check.js，默认 65% 阈值）。低于阈值 CI 不通过。
  • 契约校验：npm run contract:check（agent-event-contract.js），确保 visual-mock-server 覆盖所有真实 agent:event 类型。
